## 本轮已落实改动

- 学生榜单入口：`财商战力` 改为 `学习进度`，CTA 改为查看学习榜。
- 榜单页面：`战力排行榜` 改为 `学习进度榜`，去掉“超越前一名”“守住王座”等高压文案。
- 平台侧栏：`战力榜` 改为 `学习榜`。
- 任务权益：`波动红包` 改为 `波动观察`，`体验金` 改为 `练习资金`，`学习红包` 改为 `学习皮肤`。
- 任务中心护栏：统一为“不改变净值、学习点或学习榜”。
- 订阅边界：`BillingIntent` 拆分为 `guest-upgrade-prepay` 与 `parent-link-prepay`；`/api/billing/prepay` 现在禁止所有学生角色直接创建订单。
- 游客升级：学生界面不再直接创建付款订单，改为“绑定个人账号 -> 生成家长确认链接”。
- 价格页：学生登录后不显示微信付款按钮，只显示家长确认链接入口。
- 任务中心页：补充订阅安全横幅，让过期/基础模式学生能生成家长确认链接，而不是只看到升级错误。
- 家长确认 E2E：新增 Playwright 覆盖“学生生成确认链接 -> 学生持 token 仍不能付款 -> 家长打开 `/pricing?upgrade` -> 浏览器发起标准版人工核验订单并绑定到学生账号”。

## 验收口径

- `rg -n "红包|体验金|王座|超越|比拼|抢占头名|冲一名" src/components src/lib` 应为 0。
- 学生端可见核心路径不再把竞争排名作为唯一主反馈。
- 所有奖励文案都说明其仅用于学习记录、装饰或复盘，不影响真实收益。
- 后续每个阶段仍需执行：实现 agent 与审核 agent 分离，完成后输出 `APPROVE / REQUEST_CHANGES`。
