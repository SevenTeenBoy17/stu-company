# Brown Zone 真实用户内测问题清单（2026-07-05）

## 结论

本轮按“真实用户必须等待页面、接口或内容返回后才能进入下一步”的方式，完成了 3 个周期的模拟内测：

1. 新用户入口与游客体验路径。
2. 已登录学生核心模块路径。
3. 移动端、异常输入与回归路径。

在覆盖路径内，没有发现会阻断登录、游客进入学生端、学生核心模块访问、移动端展示或风险测评提交的 P0/P1/P2 产品问题。唯一失败标记来自“故意输错密码”场景返回的 401，这是预期行为，界面已显示主流、清晰的中文错误提示：“账号或密码错误，请重新输入。”

## 下一步建议

1. 将本轮 Playwright 流程固化为正式 E2E 测试文件，避免后续只靠人工截图回归。
2. 给登录弹窗、游客体验、学生页主标题、侧栏导航等关键节点增加稳定 `data-testid`。
3. 若后续继续扩展学生端长页面，建议每次 UI 改动后固定跑“桌面 1440px + 移动 390px”的横向溢出检查。
4. 若要进一步提升可访问性，可对隐藏文本节点做一次语义审计，确认哪些是给屏幕阅读器用，哪些只是动画/布局残留。
