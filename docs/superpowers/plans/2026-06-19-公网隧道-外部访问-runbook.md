## Step 1 — 安装 cloudflared
```powershell
winget install --id Cloudflare.cloudflared
# 或：下载 https://github.com/cloudflare/cloudflared/releases 的 cloudflared-windows-amd64.exe，放进 PATH
cloudflared --version    # 验证装好
```

## Step 2 — 起隧道，拿到公网地址
```powershell
cloudflared tunnel --url http://localhost:3000
# 输出里找这一行（每次随机）：  https://<xxxx>.trycloudflare.com   ← 这就是公网地址
# 这个窗口要一直开着（隧道随它活）
```

## Step 3 — 设 APP_URL + 生产模式起服务
1. 编辑 `.env.local`，把 `APP_URL` 改成 Step 2 的隧道地址（**必须**，否则 checkOrigin 会拒掉注册等所有写请求）：
   ```
   APP_URL=https://<xxxx>.trycloudflare.com
   ```
2. 另开一个窗口，停掉占着 :3000 的 dev server，然后生产模式起：
   ```powershell
   cd "D:\树德实验中学（清波）\C2\brown-zone-web"
   npm run build
   npm run start          # = NODE_ENV=production → secure cookie 生效 + checkOrigin 用 APP_URL
   ```

## Step 4 — 验收 + 安全复检（必须）
- 浏览器开 `https://<xxxx>.trycloudflare.com` → 能**注册新学生**、能正常用、刷新数据还在。
- **安全复检**：`superadmin` / `admin@brownzone.ai`（`BrownZone2026!`）登录应**失败**；`MRB-TEACHER-2026` 注册应**失败**。两者都失败 = 后门已堵，可放心分享地址。

## 必须知道的现实约束
- **电脑要开机，且两个窗口（cloudflared 隧道 + `npm run start` 生产服务）都得一直开着**，关任一个就断网。
- **trycloudflare 地址是临时的**：重启隧道会变 → `APP_URL` 要跟着改、生产服务要重起。
- trycloudflare 有流量/并发限制，**适合小范围试用**（几十人内测），不适合大规模生产。
- **要稳定长期地址**：用命名隧道 + 你自己的域名（`cloudflared tunnel login` → `tunnel create` → `tunnel route dns` → `config.yml`）。这条我可以单独带你配，地址固定、可上生产。

## 🔁 重启后恢复（实测可用，2026-06-24 跑通两次）

电脑一重启，Docker / 隧道 / 生产服务全断、公网地址失效；但 **DB 命名卷保留**（管理员 + 数据都在，**无需重新 seed**）。按这 6 步拉回来（全程在仓库根目录）：

