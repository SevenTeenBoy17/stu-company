### 1.3 Brown Zone 差异化定位

**核心壁垒 (竞品无法快速复制的 3 个能力)**:

1. **AI 行为诊断闭环**: 唯一将认知偏差检测 (过度交易/损失厌恶/羊群效应) 嵌入教学循环的产品 — 不是"你赚了/亏了"，而是"你 *为什么* 这样决策"
2. **12 回合叙事沙盘**: 不是无限制的模拟交易，而是有起承转合的 3 幕结构 (基础→复杂→综合)，每回合引入一个精确的金融概念
3. **三端联动 (学生-教师-家长)**: 同一套行为数据服务课堂教学和家庭反馈，国内竞品均无此闭环

**市场空白 (我们填补的缺口)**:
- 国内无一款产品同时满足: 青少年友好 + 模拟交易 + AI 个性化 + 家校协同 + 微信生态
- SIFMA 模式验证了学校渗透可行，但中国没有中文本土化版本
- 现有模拟炒股全部面向成人，无认知负荷适配和未成年人保护

## 第二部分: 四大需求深度分析与实施方案

### 需求 1: 微信收费体系 — 游客试用 + 订阅制

#### 2.1.1 深度分析

**为什么选 ¥15/月而不是更高/更低?**
- 中国青少年教育类订阅心理价位区间: ¥9.9-29.9/月
- ¥15 处于"不犹豫"区间 — 一杯奶茶的价格，家长决策压力极低
- 对标: 少年得到 ¥19.9/月、樊登小读者 ¥12/月、网易有道精品课 ¥15-25/月

**为什么选 1-3 天试用而不是 7 天?**
- 问题: 7 天试用在教育产品中转化率仅 8-12%（用户体验完核心循环后失去紧迫感）
- 建议: **1 天全功能 + 3 天 AI 基础版** (阶梯式降级，而非断崖式关闭)
  - Day 1: 全功能 (含 AI 行为诊断)
  - Day 2-3: 可继续操作，但 AI 反馈降级为通用模板 (非个性化)
  - Day 4+: 仅可查看历史记录，不能执行新操作
- 心理学依据: 禀赋效应 (endowment effect) — 学生已"拥有"了自己的投资组合，失去操作权比没有开始更痛苦

**微信支付而非支付宝?**
- 中国 K-12 家长沟通 98% 通过微信
- 微信支付+微信服务号通知 形成天然的续费提醒闭环
- 未来可延伸到微信小程序版本

#### 2.1.2 订阅分层设计

```
┌─────────────────────────────────────────────────────────┐
│                    订阅分层架构                           │
├───────────┬────────────────┬────────────────────────────┤
│  游客体验  │   个人标准版     │      校园/家庭旗舰版        │
│   免费     │   ¥15/月       │      ¥299/学期/班级         │
├───────────┼────────────────┼────────────────────────────┤
│ 1天全功能  │ 12回合完整沙盘  │ 教师端+家长端完整功能        │
│ +3天降级   │ AI个性化诊断    │ 多班级管理+班际排行          │
│           │ 历史复盘看板    │ 行为分析报告导出             │
│           │ 排行榜参与      │ 定制赛事主题                │
│           │               │ 优先客服                    │
└───────────┴────────────────┴────────────────────────────┘
```

#### 2.1.3 支付流程设计 (未成年人保护)

```
学生点击「解锁完整版」
    │
    ▼
显示家长授权弹窗
    │ "这是一个教育类订阅，需要家长完成支付"
    │ "请将此页面转发给家长，或输入家长手机号发送短信"

    │
```

- 月度自动续费需微信代扣协议签约 (支持随时取消)

### 需求 2: 邮箱注册体系

#### 2.2.1 深度分析

**现状问题**:
- 当前系统使用邀请码注册，限制了自然增长
- Demo 页面有预置账号和邀请码，对新用户不够直觉
- 没有标准的邮箱注册流程

**设计原则**:
- 邮箱为唯一标识 (一邮一户)
- 邮箱格式验证 + 重复检查 (注册时实时反馈 "该邮箱已注册")
- 密码最低 8 位 (已满足)
- 注册后自动分配游客试用身份，不需要邀请码
- 邀请码为可选字段 — 有码则加入对应班级，无码则为独立用户

#### 2.2.2 注册流程重设计

```
新用户首次访问 → 落地页 → 点击「免费体验」
    │
    ▼
注册表单 (3 字段 + 1 可选):
  ┌──────────────────────────────┐
  │  昵称 (2-16 字符)            │
  │  邮箱 (实时查重)             │
  │  密码 (≥8位, 强度指示器)      │
  │  邀请码 (可选, 有则加入班级)   │
  └──────────────────────────────┘
    │
    ▼
注册成功 → 自动登录 → 进入新手引导流程
    │
    ├─ 有邀请码 → 角色和班级由邀请码决定
    └─ 无邀请码 → 默认为"独立学生", 创建个人沙盘
```

#### 2.2.3 数据库变更

```sql
-- 新增: 允许无邀请码注册的独立用户
-- users 表已支持 classroom_id = NULL
-- 需要新增: trial_expires_at 列 (试用到期时间)
-- 需要新增: subscription_tier 列 ('free'|'standard'|'premium')

ALTER TABLE users ADD COLUMN trial_expires_at TIMESTAMP;
ALTER TABLE users ADD COLUMN subscription_tier VARCHAR(20) DEFAULT 'free';
```

### 需求 3: AI 引导式新手教学 (游戏化叙事 + 认知负荷理论)

#### 2.3.1 深度分析

**核心挑战**: 如何在 5-8 分钟内让一个从未接触过投资的 13 岁学生:
1. 理解"为什么要学这个" (动机)
2. 掌握基本操作 (能力)
3. 想继续玩下去 (留存)

**认知负荷理论 (CLT) 应用**:

| CLT 原则 | 应用方式 | 反面教材 |
|---|---|---|
| **内在负荷管理** | 每步只引入 1 个概念 | 一次性解释股票+ETF+债券+商品+外汇 |
| **外在负荷减少** | 非必要 UI 元素在教学时隐藏 | 教学时显示完整交易面板 |
| **关联负荷引导** | 每个概念配一个即时动手操作 | 纯文本说明 → 做题 |
| **工作记忆限制** | 每屏 ≤4 个选项/信息块 | 8 模块同时展示 |
| **先备知识激活** | 用生活类比引入 (校园小卖部=供需) | 直接用专业术语 |

#### 2.3.2 新手教学设计 — "Mr.Brown 的第一堂课"

**叙事线**: 学生收到一封来自 Mr.Brown 的"邀请信"，说学校获得了一笔模拟资金，需要组建投资小组来管理它。

**7 步引导流 (渐进式, 总时长 5-8 分钟)**:

```
Step 1: 「你好，新同学」(30 秒)
  - Mr.Brown 自我介绍 (AI 对话气泡, 非弹窗)
  - "我是 Mr.Brown，你的理财教练。接下来我会带你认识这个沙盘。"
  - 关键词引入: 零 (不用真钱)
  - 动作: 学生点击「开始」

Step 2: 「你有 12 万启动资金」(45 秒)
  - 显示资金面板 (只显示现金, 隐藏其他模块)
  - Mr.Brown: "想象这是你的班费基金，怎么让它在 12 周后变得更多？"
  - 关键词引入: 现金, 净值
  - 动作: 无 (纯观察)

Step 3: 「先认识两个朋友」(60 秒)
  - 只显示"股票"和"债券"两个资产 (不显示 ETF/商品/外汇)
  - Mr.Brown: "股票像坐过山车，可能涨很多也可能跌; 债券像存银行，稳但慢。"
  - 类比: 股票 = 学校运动会赌谁赢 (刺激但不确定), 债券 = 把零花钱存进储蓄罐 (安全但无聊)
  - 关键词引入: 风险, 收益, 波动
  - 动作: 无

Step 4: 「试试买入你的第一笔」(90 秒) ⚡ 关键转化步
  - 引导买入少量股票 (预填数量=10, 学生只需点确认)
  - Mr.Brown: "不用怕，这是模拟的，先感受一下什么是'下单'。"
  - 买入后立即显示: 持仓 +10 股, 现金减少, 净值不变
  - 关键词引入: 买入, 持仓, 市价
  - 动作: 点击「确认买入」

Step 5: 「时间快进」(60 秒) ⚡ 情绪锚点
  - 自动推进到第 2 回合 (市场上涨场景)
  - 净值变化动画 (数字跳动 + 绿→红 色变)
  - Mr.Brown: "看！你的组合涨了。但下一回合可能会变 — 这就是市场。"
  - 关键词引入: 回合, 涨跌, 市场变化
  - 动作: 观察 (形成 "我赚了!" 的正向情绪)

Step 6: 「认识你的第一个事件卡」(60 秒)
  - 弹出简化版事件卡 (只显示标题+信号+一句话)
  - Mr.Brown: "市场不是随机的，有新闻和政策在影响它。看看这张卡告诉了你什么。"
  - 关键词引入: 事件卡, 利好/利空, 宏观
  - 动作: 点击「我知道了」

Step 7: 「你的旅程开始了」(30 秒) ⚡ 转入正式沙盘
  - 显示完整仪表盘 (此时所有模块解锁)
  - Mr.Brown: "接下来的 10 个回合，你可以自由决策。每几回合我会给你一份行为报告。加油！"
  - 关键词引入: 无新概念 (巩固前 6 步)
  - 动作: 开始正式沙盘
```

#### 2.3.3 渐进式概念解锁表

| 回合 | 解锁概念 | 解锁资产/功能 | 教学方式 |
|---|---|---|---|
| 教学流 | 现金, 净值, 风险/收益, 买入 | 股票, 债券 | AI 引导对话 |
| 1-2 | 分散投资, 持仓比例 | ETF | 事件卡提示 |
| 3-4 | 止盈止损, 机会成本 | 储蓄/贷款 | 行为诊断推送 |
| 5-6 | 供需关系, 对冲思维 | 商品 | 回合复盘 |
| 7-8 | 汇率, 全球联动 | 外汇 | AI 教练点评 |
| 9-10 | 危机管理, 现金缓冲 | 高波动事件 | 压力测试 |
| 11-12 | 长期视角, 再平衡 | 全功能回顾 | 成长报告 |

### 需求 4: 随机金融事件系统升级

#### 2.4.1 深度分析

**现状**: 当前有 6 张事件卡，12 回合中部分重复使用。事件类型覆盖 macro/sentiment/policy，但缺少:
- 微观企业事件 (财报、产品发布)
- 国际联动事件 (汇率干预、贸易摩擦)
- 黑天鹅事件 (极端但有教育价值的场景)
- 学生自身的"决策复盘"事件 (基于行为触发)

**设计原则**:
1. **教育优先**: 每个事件必须有明确的教学目标
2. **难度递进**: Round 1-4 事件影响 ≤±8%, Round 5-8 ≤±15%, Round 9-12 ≤±22%
3. **因果明确**: 13 岁学生能理解 "为什么市场涨/跌了"
4. **平衡性**: 不存在单一策略在所有事件下都最优

#### 2.4.2 事件卡扩展分类

```
事件卡分类体系 (从 6 张 → 24 张)

├── 宏观经济 (6 张)
│   ├── 消费回暖 (利好, R1-4)
│   ├── 通胀压力 (利空, R5-8)
│   ├── 央行降息 (利好, R5-8)
│   ├── 全球衰退预警 (利空, R9-12)
│   ├── 经济复苏确认 (利好, R9-12)
│   └── 供应链中断 (中性, R5-8)
│
├── 政策调控 (6 张)
│   ├── 监管趋严 (利空, R3-6)
│   ├── 产业扶持政策 (利好, R1-4)
│   ├── 房产调控加码 (利空, R7-10)
│   ├── 绿色能源补贴 (利好, R5-8)
│   ├── 数据安全新规 (中性, R7-10)
│   └── 教育改革利好 (利好, R1-4)
│
├── 市场情绪 (6 张)
│   ├── 并购潮 (利好, R3-6)
│   ├── 恐慌性抛售 (利空, R9-12)
│   ├── FOMO 情绪蔓延 (中性, R5-8)  ← 教学点: 羊群效应
│   ├── 价值回归 (利好, R9-12)
│   ├── 信息茧房效应 (中性, R7-10)  ← 教学点: 确认偏差
│   └── 市场分化加剧 (中性, R5-8)
│
├── 行业竞争 (4 张)
│   ├── 科技龙头财报超预期 (利好, R3-8)
│   ├── 行业替代效应 (中性, R5-10)
│   ├── 新能源产能过剩 (利空, R7-12)
│   └── 消费升级趋势 (利好, R1-6)
│
└── 黑天鹅 (2 张, 低概率高影响, 仅 R9-12)
    ├── 流动性危机 (利空, 全资产 -10~-18%)
    └── 意外政策转向 (利好, 全资产 +8~+15%)
```

#### 2.4.3 自适应事件机制 (创新卖点)

**基于学生行为的动态事件**: 除固定事件外，AI 根据学生的操作历史生成"教学干预事件":

| 检测到的行为 | 触发的教学事件 | 教学目标 |
|---|---|---|
| 连续 3 回合只持有股票 | "市场调查显示: 你班里 60% 的同学也全仓股票" | 羊群效应反思 |
| 从未使用过债券 | "新闻: 近期债券收益率创新高" | 引导尝试分散 |
| 3 回合内交易 > 9 次 | "交易所提醒: 频繁交易可能增加成本" | 过度交易警示 |
| 亏损后大幅加仓 | "Mr.Brown 提醒: 这叫'报复性交易'" | 损失厌恶教学 |
| 净值持续上涨 | "恭喜！但别忘了设置你的'舒适回撤线'" | 止盈/风控意识 |

## 第三部分: 产品创新卖点与市场差异化

### 3.1 三大核心卖点

**卖点 1: "AI 读懂你的投资性格"**
- 不是告诉你赚了还是亏了，而是告诉你 *你是什么类型的投资者*
- 6 维雷达图: 风控意识 / 资产配置 / 交易节奏 / 宏观视野 / 复盘能力 / 目标执行力
- 每月生成《投资行为成长报告》，家长可通过微信接收

**卖点 2: "12 回合叙事沙盘，不是无脑炒股"**
- 有剧情、有事件卡、有节奏感的"投资养成游戏"
- 每回合 = 一个教学单元，从基础概念到黑天鹅应对
- 适合 45 分钟课堂嵌入，也适合个人自主推进

**卖点 3: "一杯奶茶的价格，给孩子一学期的财商课"**
- ¥15/月 = ¥45/学期，远低于线下财商课 (¥500-2000/期)
- 家长通过微信支付，3 秒完成
- 30 天无理由退款 (降低决策门槛)

### 4.3 关键技术约束

1. **支付安全**: WeChat Pay APIv3 签名验证，商户密钥仅在 env 中，金额用整数分
2. **未成年人保护**: 学生端不展示金额，支付必须经家长账号
3. **AI 网关**: 所有 AI 调用走 `src/lib/ai.ts`，不增加新的 provider fetch
4. **数据隔离**: billing schema 与 public schema 分离
5. **渐进增强**: 所有新功能在无微信支付配置时优雅降级 (同现有 DB/AI 降级模式)
