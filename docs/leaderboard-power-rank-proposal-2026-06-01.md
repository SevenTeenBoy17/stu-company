# 财商战力排行榜系统 · 产品设计与升级方案（待 review）

> **产品**：Mr.Brown AI 经济沙盘
> **目标用户**：12–18 岁中学生
> **角色视角**：资深教育游戏产品设计与制作
> **状态**：设计方案（本轮不实现，待你 review 后再开发）
> **日期**：2026-06-01

---

## 0. 一句话定位

把现在的「周赛季净值榜」升级为 **「财商战力 × 地域段位榜」**——像《王者荣耀》战力那样，让每个学生在 **校内 / 地级市 / 省 / 全国 × 周 / 月 / 学期赛季** 的多维榜单里看到自己的位置和成长。
**但排的不是"谁更敢赌"，而是"谁的财商决策更稳更强"**——这是本方案与一个普通净值榜最本质的区别。

## 1. 需求理解与重述（含我读到的言外之意）

你给的显性需求：
- 学生**自行输入学校单位 + 地区**；
- 做成 **rank 排行榜**，参考王者荣耀**战力排行**；
- 维度：**地级市 / 全省 / 全国**；
- 周期：**周榜 / 月榜 / 赛季榜（一学期 = 一个赛季）**。

我读到的**隐性需求 / 关键洞察**（资深教育游戏设计师视角）：
1. **排行榜的"排什么"= 你在"教什么"。** 这是整个功能最重要的一处设计决策。若按"净值"排，学生会学到"梭哈高风险+赌随机事件中奖=第一"，与财商教育目标背道而驰。所以战力分必须**奖励好决策（稳健、纪律、风险调整后收益）而非运气**。
2. **王者战力的真正魅力不是"全国第一"，而是"人人有段位、人人有梯子"。** 全国榜只激励前 1% ，会打击 99% 。必须配 **段位系统 + 校内榜/同城榜**，让每个学生都有够得着的目标。
3. **12–18 是未成年人**：学校 + 地区 + 真实姓名 = 可识别 + 可定位 → 强隐私/合规约束（PIPL、未成年人网络保护条例）。自填数据还有**真实性 / 内容安全 / 防作弊**问题。
4. 已有基建可复用：`src/lib/season.ts`（周赛季确定性种子，保证"同种子=同市场=可公平横比"）、`buildSeasonLeaderboard`、物化 `net_worth` 列 + 复合索引、`/api/market/season-leaderboard`、Vercel weekly cron、班级/邀请码体系、persona 分享。**不是从零做，是站在现有肩膀上扩展。**

## 2. 设计红线（12–18 教育产品 + 中国合规，先立后破）

| 红线 | 说明 |
| --- | --- |
| **不能让"高风险博弈"成为榜单最优解** | 核心教育红线。战力分必须做风险调整 + 纪律加权（见 §3）。 |
| **未成年人隐私** | 公开榜**绝不**同时暴露「真实姓名 + 学校 + 地区」（可被人肉）。公开只显示**昵称 + 段位 + 战力 + 名次**，学校可选脱敏（"某市某中学"或仅校内可见）。 |
| **合规收集** | 收集地区/学校属未成年人个人信息，需**告知 + 监护人同意**（已有家长端可承载）；遵循**数据最小化**——只收到"地级市 + 学校"足够排名，不强制区/县/详细地址。 |
| **健康竞争 / 防沉迷** | 不羞辱垫底（垫底不公示真实身份）、重过程轻名次（连胜/最大进步等正向激励）、**无付费买榜**、无赌博美化、控制刷榜驱动的过度使用。 |
| **可学性** | 战力构成对学生**透明**——"为什么他战力高"本身就是一堂财商课。 |

## 3. 核心设计 ①：财商战力分（Financial Power Score）

### 3.1 为什么不能只排净值
净值榜 = 鼓励梭哈 + 押随机事件。一个赌赢的学生战力第一 → 全校模仿 → 教育失败。**这是必须改的第一件事。**

### 3.2 财商战力 = 决策质量的合成分（建议 0–2000 战力值，对标王者量级）

```
战力 = 归一化(  w1·风险调整后收益       (Sharpe 式：收益 ÷ 承担的风险)
              + w2·纪律分               (不过度交易/不报复性交易/有现金缓冲，已有 disciplineScore)
              + w3·回撤控制             (最大回撤越小越高，已有 maxDrawdown)
              + w4·学习完成度           (完成复盘/读 AI 反馈/认真做决策卡)
              + w5·本金增长             (终值增长，但权重最低，避免回到"唯净值")
            )  → 映射到 0–2000
```

- **现有数据已支持**：`netWorth / riskScore / disciplineScore / maxDrawdown` 在历史复盘里都有；学习行为可由动作日志统计。
- **权重 `w1..w5` 待你拍板**（我的初始建议：30/25/20/15/10，风险调整收益与纪律为主）。
- **透明可学**：战力卡上展示构成雷达（你已有 6 维行为雷达可复用），学生一眼看懂"我哪块强、哪块该补"。

> 教育游戏精髓：**让计分规则 = 课程目标**。学生为了上分，自然去学"控制风险、保持纪律"，而不是"梭哈"。

## 4. 核心设计 ②：段位体系（人人有梯子）

对标王者"青铜→王者"，主题化为财商成长阶梯：

| 段位 | 主题名（建议） | 战力区间(示意) | 心智 |
| --- | --- | --- | --- |
| 1 | 理财新手 🥉 | 0–399 | 起步，重在完赛 |
| 2 | 稳健学徒 🥈 | 400–799 | 学会留现金缓冲 |
| 3 | 精明投资者 🥇 | 800–1199 | 风险收益平衡 |
| 4 | 策略大师 💎 | 1200–1599 | 穿越事件周期 |
| 5 | 财商宗师 👑 | 1600–1899 | 稳定高分 |
| 6 | 巅峰·名人堂 ✨ | 1900+ | 赛季顶尖 |

- **解决"榜单只激励前 10%"**：每个学生都有**专属晋升进度条**，与全国名次解耦——哪怕全国第 8000 名，也能看到"距离下一段位还差 60 战力"。
- **对未成年人更温和**：可设**段位保底**（不掉段或只软掉），避免挫败；升段有庆祝动画（你已用 framer-motion）。

## 5. 核心设计 ③：多维榜单矩阵

### 5.1 地域维度（王者式"市/省/全国第 X"）
- **校内榜**（最具激励 ← 和认识的同学比）
- **地级市榜**
- **全省榜**
- **全国榜**

战力卡同时显示四个名次：
> 你的财商战力 **1,460** · 策略大师 💎
> 🏫 树德实验中学 **第 1 名** · 🏙️ 成都市 **第 12 名** · 🗺️ 四川省 **第 340 名** · 🇨🇳 全国 **第 5,231 名**

（即便全国 5000 名，"校内第 1 / 市第 12"也足够有成就感——这是分层榜单的意义。）

### 5.2 时间维度
- **周榜**：每周刷新（直接复用现有周赛季种子机制），高频参与。
- **月榜**：当月聚合。
- **赛季榜（一学期）**：头牌榜，学期长周期 + **赛季结算 + 段位继承（软重置）+ 赛季奖励**。

### 5.3 学期赛季节奏（建议，待你按校历确认）
- **春季赛季**：约 2 月中 – 6 月底
- **秋季赛季**：约 9 月初 –次年 1 月底
- 赛季末：结算名次 → 发赛季奖励（专属头衔/边框/战力卡皮肤，**纯荣誉、不付费**）→ 段位软重置（高段位继承一部分，像王者继承星）。

## 6. 学校 / 地区数据：自填 + 治理 + 防作弊

### 6.1 采集方式（结构化，不要自由文本）
- **地区**：省 → 市（→ 可选区/县）三级**下拉选择器**，基于国标 GB/T 2260 行政区划数据集（约 34 省 / 333 地级市 / 2800+ 区县，开源静态数据可直接导入）。
- **学校**：**自动补全 + 众包待审**——输入时从学校库联想；库里没有则提交"待审"，审核/去重后入库（避免"成都七中"/"成都市第七中学"/"七中"三条重复）。

### 6.2 可信分级（关键）
- **班级绑定（高可信）**：学生用教师发的**邀请码**加入班级时，学校由**教师**确定 → 可信。优先用这个反推学校。
- **自填（低可信）**：未加入任何班级的个人学生自填 → 标记为"自报"，可放**单独的"个人挑战榜"**或在校榜里弱化，防止刷校榜。
- 这样既满足你"学生可自填"的需求，又给真实性留了闸门。

### 6.3 防作弊
- 一人多号刷校榜：设备/IP 限流（已有 rate-limit）、同校异常增长检测、新号需完赛若干回合才计入榜。
- 战力分本身**抗刷**：因为它奖励稳健决策而非单局净值，"开小号赌一把上分"在合成分下收益很低。
- 昵称/学校提交**内容安全审核**（敏感词 + 人工抽审）。

## 7. 隐私与合规设计（未成年人）

- **公开榜可见**：昵称、段位、战力值、名次、（可选）学校名。**不可见**：真实姓名、班级、详细地区、任何可定位到个人的组合。
- **家长告知/同意**：首次启用"上榜"前，走告知 + 监护人同意（复用家长端）；提供**一键关闭上榜 / 仅校内可见 / 完全匿名**三档可见性（opt-in）。
- **数据最小化**：排名只需"地级市 + 学校"，不强制收集区县/地址/学籍号。
- 注：具体合规口径建议过一遍法务（PIPL + 未成年人网络保护条例 2024 生效）。

## 8. 技术架构与数据模型（在现有基础上扩展，非重写）

**复用现有**：`season.ts`（周种子=同市场=可公平横比）、`buildSeasonLeaderboard` / `getSeasonLeaderboard`、物化 `net_worth` 列 + 复合索引、`/api/market/season-leaderboard`、weekly cron、classroom/invite。

**新增**：
1. **数据**：`profile` 增 `province_code / city_code / district_code? / school_id / region_source(classroom|self) / leaderboard_visibility`；新 `schools` 表（含 `status: approved|pending`）；新 `power_score`（物化，每 run/用户/周期）；`seasons` 元数据表（学期赛季）；导入**行政区划**静态数据集。
2. **战力计算**：纯函数 `computePowerScore(runMetrics)`（可单测，复用现有 TDD 习惯）。
3. **排行服务**：SQL 窗口函数 `RANK() OVER (PARTITION BY scope, period ORDER BY power DESC)`；**周期性预计算 + 缓存**（复用 cron：周榜每日刷、月/赛季按需），分页。
4. **公平性**：同赛季种子=同市场（已有）→ 跨人可比。
5. **扩展性**：物化 + 索引 + 缓存 + 分页，应对规模增长。

> 工程量上，V1 主要是"在周赛季榜外面套 地域分区 + 战力分 + 段位"，**复用度很高**。

## 11. 风险与权衡

- **战力公式调参**：权重不当会再次跑偏（如净值权重过高→又回到赌）。建议先小范围灰度 + 看分布。
- **冷启动**：早期人少，市/省/全国榜稀疏。对策：先重**校内榜 + 周榜**，地域榜随规模解锁；或用段位（绝对值）而非名次先撑住成就感。
- **自填数据噪声**：靠"班级绑定优先 + 众包待审 + 抗刷战力分"控制。
- **合规**：未成年人数据是高压线，需法务确认 + opt-in。

## 12. 需要你 review 拍板的关键决策

1. **战力 = 净值 还是 财商战力合成分？**（我强烈建议合成分）权重 `w1..w5` 用我的初始建议(30/25/20/15/10) 还是你定？
2. **学校数据可信策略**：班级绑定优先 + 自填走"个人挑战榜"——接受吗？还是允许自填直接进校榜（风险更高）？
3. **隐私公开度**：公开榜显示"昵称+段位+战力+名次（+可选学校）"、隐藏真实姓名——接受吗？是否需要监护人同意闸门 + opt-in 可见性？
4. **赛季日期**：按学期（春 2–6 / 秋 9–1）——具体起止以哪个校历为准？
5. **规模预期**：预计多少学生/学校？（决定是否现在就上缓存/预计算与地域分区粒度）
6. **免费 vs 付费**：排行榜是**免费**功能（建议，扩大池子 + 教育普惠），还是与订阅挂钩？赛季奖励确认是**纯荣誉无付费**吧？
7. **段位降段**：对未成年人是否设"不掉段/软掉段保底"？
