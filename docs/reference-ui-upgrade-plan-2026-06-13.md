# Brown Zone 多元投资理财教学化升级计划

日期：2026-06-13  
角色视角：资深项目经理 / 全栈开发工程师 / 青少年金融素养产品设计师  
配套问题清单：`docs/reference-ui-issues-2026-06-13.md`

## 1. 总目标

把 Brown Zone 从“学生经济沙盘 + 多个功能页面”升级为“青少年多元投资理财学习操作系统”。

目标不是复制支付宝或券商 App，而是吸收其成熟的信息架构，再转译为适合 14-19 岁学生的教育模拟体验。

核心目标：

- 更现实：覆盖行情、自选、基金/ETF、持仓、目标账户、现金流、信用、保险、风险保护、机会识别。
- 更教学：每个指标都有概念解释，每个操作都有复盘价值。
- 更好玩：任务、赛季、徽章、班级共识和观察单让学生愿意回来。
- 更安全：不做真实投资建议，不做开户导流，不承诺收益。
- 更商业化：免费体验能理解价值，订阅用户获得更完整 AI 复盘、赛季内容和报告能力。

## 2. 产品信息架构

### 2.1 四大一级域

| 一级域 | 路由 | 用户问题 | 核心体验 |
| --- | --- | --- | --- |
| 首页 | `/student` | 今天我该做什么？ | 今日任务、财富快照、赛季入口、服务宫格 |
| 行情 | `/student/market` | 市场现在发生了什么？ | 指数、温度计、自选、板块、同学热度、AI 解读 |
| 机会 | `/student/opportunity` | 哪些主题值得观察？ | 长期机会、阶段热点、观察单、证据链训练 |
| 持有 | `/student/wealth` | 我的配置现在健康吗？ | 总资产、收益拆解、持仓、目标、保护伞、历史入口 |

### 2.2 二级服务分组

| 分组 | 包含模块 | 设计目的 |
| --- | --- | --- |
| 策略行动 | 策略总览、交易面板、市场雷达、机会训练 | 训练决策，不鼓励冲动交易 |
| 资产成长 | 我的财富、基金实验、定投机器人、风险测评 | 建立长期配置和分散意识 |
| 生活理财 | 生活账本、目标账户、保护伞、信用实验室 | 把投资和真实生活目标连接 |
| 学习留存 | 任务中心、历史复盘、战力榜、课程模块 | 把行为变成学习证据和留存动力 |

### 2.3 首页服务台原则

首页首屏只回答三个问题：

1. 今天市场温度如何？
2. 我的财富状态如何？
3. 今天完成哪个学习动作最有价值？

不要把所有功能平铺成菜单。更多入口放在服务宫格和“更多服务”里。

## 4. 前端升级计划

### Phase F1：首页服务台再整理

目标：

让 `/student` 成为学生每天打开后的主入口，而不是一个复杂仪表盘。

涉及文件：

- `src/app/(platform)/student/page.tsx`
- `src/components/student/student-home-hub.tsx`
- `src/components/platform/platform-layout.tsx`
- `src/lib/student-service-map.ts`

改造内容：

- 首屏固定展示市场温度、财富快照、今日任务。
- 服务宫格按四类分组：策略、资产、生活、学习。
- 移动端优先展示四大一级域，不展示过多二级入口。
- 每个入口附一句“你会学到什么”。
- 加入“今日必看”卡片：一条市场事实 + 一个教学问题 + 一个可执行动作。

验收标准：

- 390px 手机首屏 1.5 屏内能看到今日任务、财富快照和至少 4 个入口。
- 所有入口点击后能进入真实页面或明确显示“即将开放”。
- 首页没有英文错误、没有横向滚动。

### Phase F2：持有总入口升级

目标：

把 `/student/wealth` 做成参考图“基金持有页”的学生版。

涉及文件：

- `src/app/(platform)/student/wealth/page.tsx`
- `src/components/student/student-wealth-dashboard.tsx`
- `src/lib/wealth-review.ts`
- `src/app/api/student/wealth-summary/route.ts`

页面结构：

1. 财富总览：当前净值、本回合变化、累计模拟收益、现金安全垫。
2. 收益拆解：持仓收益、储蓄收益、房产/创业变动、负债成本。
3. 资产配置：现金、储蓄、持仓、房产、创业、债务。
4. 投资计划：定投机器人、目标账户、保护伞状态。
5. 最近动作：交易、观察单、基金实验、保护伞预案。
6. AI 复盘：三段式建议。

前端要求：

- 金额用 `MoneyText` 或统一金额组件。
- 收益/涨跌采用中国金融习惯：上涨红、下跌绿。
- 移动端用短卡片，不在首屏堆大图表。
- “查看历史复盘”放在财富页明显位置。

后端要求：

- `wealth-summary` 聚合已有 `scenario_runs`、`snapshots`、`holdings`、`actionLog`。
- DB 不可用时返回可恢复中文错误。
- 不新增表，优先使用现有数据。

### Phase F3：市场雷达教学化

目标：

让 `/student/market` 从行情看板升级为“市场观察训练页”。

涉及文件：

- `src/app/(platform)/student/market/page.tsx`
- `src/components/student/student-market-board.tsx`
- `src/components/student/market-thermometer.tsx`
- `src/lib/market-sentiment.ts`
- `src/lib/student-watchlist.ts`
- `src/lib/peer-heat.ts`

新增/强化内容：

- 市场温度计：热、温、冷，并解释“市场情绪不等于买卖信号”。
- 每日必看：一条事实 + 一个概念 + 一个观察问题。
- 自选观察池：加入/移除/补充观察理由。
- 板块热度：显示热度来源和反思提醒。
- 同学热度：只展示聚合数量，不展示个人隐私。

AI 解读按钮：

- 对当前股票/板块解释。
- 输出结构固定为：观察到了什么、为什么重要、下一步模拟动作。
- 不输出“买入/卖出保证式结论”。

验收标准：

- 切换股票时主卡、雷达、关键字段、AI 上下文同步。
- 加入观察池后历史和任务进度能看到。
- AllTick 不可用时显示“教学观察池模式”，页面不空白。

### Phase F4：机会训练场闭环

目标：

把参考图中的“机会/值得投/板块热点”转译为学生写观察单的训练场。

涉及文件：

- `src/app/(platform)/student/opportunity/page.tsx`
- `src/components/student/student-opportunity-dashboard.tsx`
- `src/lib/opportunity.ts`
- `src/app/api/student/opportunity/route.ts`

核心交互：

1. 选择主题机会。
2. 选择观察理由：资金、政策、估值、风险释放、学习。
3. 写一句自己的证据。
4. 提交观察单。
5. AI 给观察质量反馈。
6. 历史复盘和赛季任务能召回这条记录。

教学约束：

- 默认按钮叫“记录观察”或“开观察单”，不叫“买入”。
- 置信度不是胜率，只是学生对自己证据链的信心。
- 热门主题必须配反思提示。

验收标准：

- 提交观察单写入 `actionLog`。
- 首页今日任务和历史复盘能识别观察单。
- 文案不包含真实荐股承诺。

### Phase F5：基金/ETF 实验室打磨

目标：

把参考图中的基金市场、基金自选、基金持有转译为“分散投资实验室”。

涉及文件：

- `src/app/(platform)/student/fund-lab/page.tsx`
- `src/components/student/student-fund-lab-dashboard.tsx`
- `src/lib/fund-lab.ts`
- `src/app/api/student/fund-lab/route.ts`

核心功能：

- 虚拟基金池：宽基成长 ETF、债券锚、黄金防守、全球科技。
- 组合方案：防守、均衡、成长。
- 指标：预期收益、风险评分、最大回撤、分散度。
- 学生记录：为什么选择这个组合。
- 历史复盘：查看过去选择和理由。

前端优化：

- 选择组合后应立即刷新预览指标。
- 金额变更后可手动预览。
- 历史记录去重并清理乱码旧记录。
- 移动端把组合方案做成横向卡片，不形成超长纵向表格。

后端优化：

- `GET /api/student/fund-lab?plan=&amount=` 支持预览。
- `POST /api/student/fund-lab` 支持 `simulate` 和 `record`。
- 记录写入 `actionLog`，不新增表。

验收标准：

- 成长组合风险和回撤高于防守组合。
- 记录后历史里能看到中文理由。
- 历史复盘能召回基金实验动作。

### Phase F6：生活目标与保护伞联动

目标：

让学生理解“理财不是只追收益，而是服务生活目标和风险承受”。

涉及文件：

- `src/app/(platform)/student/goal-accounts/page.tsx`
- `src/components/student/student-goal-accounts-dashboard.tsx`
- `src/app/(platform)/student/protection/page.tsx`
- `src/components/student/student-protection-umbrella-dashboard.tsx`
- `src/lib/goal-accounts.ts`
- `src/lib/protection-umbrella.ts`

核心联动：

- 目标账户：应急金、电脑、研学、创业。
- 保护伞：应急金、保险、债务、收入、分散、复盘纪律。
- 风险事件：手机损坏、家庭支出、市场回撤、创业现金流紧张。

教学设计：

- 先能扛风险，再谈收益。
- 目标进度受到现金流和投资波动影响。
- 保险和现金垫不是收益工具，是风险转移和缓冲工具。

验收标准：

- 保护伞预案记录后能显示在历史复盘。
- 目标账户动作能影响财富页“生活目标”区域。
- AI 建议不能鼓励学生为了收益牺牲应急金。

### Phase F7：赛季、任务和留存系统

目标：

把参考图的活动、红包、ETF 大赛转译为教育友好的留存系统。

涉及文件：

- `src/components/student/student-quest-dashboard.tsx`
- `src/lib/quests.ts`
- `src/lib/season-challenges.ts`
- `src/app/api/student/quests/route.ts`
- `src/app/api/student/season/route.ts`

任务类型：

- 今日必看：阅读一条市场事实并回答一个问题。
- 观察单：提交一条机会观察。
- 分散练习：配置不超过某个集中度。
- 保护伞：完成一次突发事件预案。
- 复盘：解释一次亏损或错过机会。

奖励边界：

- 只给徽章、称号、头像框、故事片段。
- 不直接增加净值。
- 不直接增加战力分。
- 不购买真实金融产品。

验收标准：

- 每个任务都由真实行为触发。
- 奖励领取幂等，不可重复刷。
- 教师/家长报告能看到学习行为而不是只看收益。

## 5. 后端与数据计划

### 5.1 短期：actionLog-first

优先使用 `scenario_runs.actionLog` 记录新增学习动作。

建议 action 类型 / meta：

| 行为 | 建议记录 |
| --- | --- |
| 加入观察池 | `type: "watchlist", meta.kind: "watchlist_action"` |
| 机会观察单 | `type: "opportunity", meta.kind: "opportunity_note"` |
| 基金实验 | `type: "fund_lab", meta.kind: "fund_lab_action"` |
| 目标账户 | `type: "goal", meta.kind: "goal_account_action"` |
| 保护伞 | `type: "protection", meta.kind: "protection_review"` |
| 赛季奖励 | `type: "quest", meta.kind: "season_reward_claim"` |

优点：

- 不立即新增数据库迁移。
- 离线演示仍可运行。
- 历史复盘可统一读取。
- 适合快速验证玩法是否有留存价值。

### 5.2 中期：独立表正式化

当行为数据验证有效后，再拆表：

| 表 | 用途 |
| --- | --- |
| `student_watchlists` | 学生观察池 |
| `opportunity_notes` | 机会观察单 |
| `fund_lab_records` | 基金实验记录 |
| `goal_accounts` | 生活目标账户 |
| `protection_reviews` | 风险保护伞预案 |
| `season_rewards` | 赛季奖励 |

所有表必须具备：

- `user_id` 外键
- `classroom_id` 或可推导班级
- `created_at`
- RLS 策略
- 教师/家长可见范围约束

### 5.3 API 统一规范

所有新增和继续打磨的 API 必须遵守：

- `requireUser("student")` 或明确角色权限。
- zod 校验请求体和 query。
- 错误形状统一为 `{ error, message }`。
- 写操作检查 origin / CSRF 边界。
- 外部 AI 只通过 `src/lib/ai.ts`。
- 行情接口必须字段级 fallback。
- 失败时返回中文可恢复提示，不让页面白屏。

## 6. AI 导师设计

### 6.1 统一输出协议

所有学生端 AI 反馈统一为：

```text
我观察到了什么：
...

为什么重要：
...

下一步模拟动作：
...
```

### 6.2 页面上下文

| 页面 | AI 目的 |
| --- | --- |
| 首页 | 提醒今天最值得做的一个学习动作 |
| 市场 | 解释行情指标，不给买卖结论 |
| 机会 | 评价观察理由是否有证据链 |
| 持有 | 解释资产配置和现金垫 |
| 基金实验 | 对比风险、回撤和分散度 |
| 保护伞 | 解释风险缓冲是否足够 |
| 历史复盘 | 总结行为模式和下一步改进 |

### 6.3 认知负荷原则

面向 14-19 岁学生，每次 AI 反馈最多引入一个新概念，例如：

- 最大回撤
- 分散投资
- 现金安全垫
- 机会成本
- 资金流
- 风险收益比
- 行为偏差

不要在一段反馈里同时解释过多术语。

## 7. 游戏化与教学设计

### 7.1 叙事线

建议采用“Mr.Brown 城市理财实验室”叙事：

- 市场像天气，会冷会热。
- 资产像背包，有现金、储蓄、持仓、目标和保护伞。
- 机会像线索，要先观察再行动。
- 赛季像课堂项目，看谁能解释得更清楚，而不是谁赌得更大。

### 7.2 任务循环

每日核心循环：

```text
看市场温度 -> 读今日必看 -> 写一个观察或做一个模拟配置 -> 推进回合 -> 看 AI 复盘 -> 领取学习徽章
```

每周核心循环：

```text
完成赛季主题 -> 班级共识比较 -> 历史复盘 -> 教师点评 -> 家长报告摘要
```

### 7.3 竞争设计

排行榜不只看收益，建议拆成：

- 决策纪律
- 分散配置
- 学习完成
- 复盘质量
- 风险保护
- 净值表现

这样可以避免学生只追高风险收益。

## 8. 商业模式处理

### 8.1 免费体验

免费用户应能体验：

- 首页服务台
- 一部分市场观察
- 一次机会观察单
- 一次基金实验
- 基础历史复盘

目的：

让用户理解产品价值，而不是只看到付费墙。

### 8.2 订阅用户

订阅权益建议：

- 完整 AI 个性化复盘
- 更多赛季主题
- 历史报告导出
- 家长成长报告
- 班级/学校挑战赛
- 更多基金/目标/保护伞场景

注意：

订阅不能让学生在排行榜中获得不公平分数，也不能直接提高模拟收益。

### 8.3 学校授权

学校端权益：

- 批量账号
- 班级任务
- 教师点评
- 班级数据看板
- 学习报告
- 校际挑战赛

这是比个人订阅更稳定的商业路径，应在后台运营面板中优先体现。

## 9. 测试计划

### 9.1 自动化测试

单测：

- 市场温度计算
- 自选观察池状态
- 机会观察评分
- 基金组合风险/收益/回撤
- 目标账户进度
- 保护伞压力测试
- 赛季任务进度

API 测试：

- 未登录访问返回 401
- 非学生访问学生 API 返回 403
- 请求体错误返回 400
- DB 不可用返回中文 503
- 重复提交不重复发奖励

E2E：

- 新学生登录进入首页服务台
- 加入观察池
- 提交机会观察单
- 切换基金方案并记录
- 设置目标账户
- 完成保护伞预案
- 推进回合
- 历史复盘看到行为记录
- AI 解释不含真实荐股承诺

### 9.2 视觉验收

视口：

- `390x844`
- `768x1024`
- `1440x980`

检查项：

- 无横向滚动。
- 关键数字不溢出卡片。
- 移动端首屏有明确主任务。
- 按钮有 hover / focus / loading / disabled / success / error。
- 长图表不在手机首屏强行展示。
- 金额和涨跌颜色一致。
- AI 浮窗不遮挡主操作。

### 9.3 内容安全

扫描禁用表达：

- 保证收益
- 一定上涨
- 现在必须买
- 真实开户
- 无风险赚钱
- 稳赚

替代表达：

- 教育模拟
- 可以观察
- 风险提示
- 下一步验证动作
- 写下你的理由

## 11. 推荐下一步

下一轮建议不要再做大范围重构，而是按“一个切片、一个闭环”的方式推进：

1. 先把基金实验室修成选择组合即刻预览，并让记录进入历史复盘。
2. 再把机会训练场记录接入首页今日任务和历史复盘。
3. 然后把财富页聚合目标账户、保护伞、定投和最近动作。

这样每一步都有可见价值，也不会把系统推向不可控的大改。

## 12. 项目经理终审

本计划与参考图的关系是“结构借鉴、教育转译、商业克制”。参考图的强营销按钮不适合直接进入学生端，但其信息层级非常值得学习。Brown Zone 下一阶段应把现实理财的复杂性压缩为学生可操作的学习任务，并通过 AI 解释和历史复盘形成闭环。

终审结论：可以执行，建议从 M1-M5 开始，优先做入口收敛和行为闭环。
