# Brown Zone 内测后优化升级计划（2026-06-29）

## 目标

把 Brown Zone 从“功能已经比较丰富的教育沙盘”推进到“可课堂试点、可家长理解、可持续订阅的青少年财商游戏化产品”。原则是：

- 先教育，再游戏化；游戏化只服务学习行为。
- 先解释理由，再允许行动；不鼓励冲动交易。
- 先课堂可执行，再追求大而全。
- 面向未成年人，避免抽卡、排名和真实投资暗示。

## Phase A：青少年护栏与文案去风险化

### 交付

- 将所有“战力/王座/超越/抽卡/稀有/补抽”等高刺激词汇替换为“学习进度/复盘区间/领取收藏/概念徽章”。
- 学生端付费提示统一为“家长确认/开通说明”，不在学生界面使用“付款链接”作为主要行动词。
- 在交易、AI 建议、排行、市场观察、任务奖励附近补充就地风险提示。
- 给关键操作增加“为什么这样做”的一句解释输入或选择。

### 验收

- `rg -n "抽卡|稀有|补抽|王座|超越|战力差距" src` 结果为 0 或只在历史文档中出现。
- 学生端主流程不再把排名/收益作为唯一主反馈。
- 新手 5 分钟内只遇到 1 个核心概念和 1 个下一步动作。

## Phase B：任务中心游戏体验二次打磨

### 交付

- 翻卡交互扩展到任务详情、卡库、伙伴图鉴之间的连续路径，并确保每次翻面后焦点进入当前可见卡面。
- 奖励后出现“下一步任务建议”，例如“去市场雷达写一条观察理由”。
- 卡牌背面只放角色、进度和行动动词；正面再展示学习目标、导师提示和奖励。
- 移动端降低无限浮动动画数量，支持 reduced-motion。

### 验收

- `/student/quests` 从打开卡背到领取收藏卡的路径有自动化测试。
- 翻面后隐藏卡面不可被 Tab 命中；可见卡面的第一个核心操作自动获得焦点。
- 移动端无横向滚动；低动效模式无强制 smooth scroll。
- 每张任务卡的文字不超过 2 行摘要，详细说明进入展开层。

## Phase C：课堂试点闭环

### 交付

- 教师端新增课堂模式：课前发布 1 个目标、课中干预队列、课后出口卡。
- 学生端任务与教师发布目标关联。
- 班级报告以“完成率、复盘质量、风险行为、优秀解释样例”为主，不以净值排名为主。

### 验收

- 30 人以内班级，教师 5 分钟内能定位未完成任务和需干预学生。
- 45 分钟课堂流程可闭环：概念导入 -> 沙盘行动 -> 小组解释 -> 出口卡。
- 教师端可导出或复制一页课后总结。

## Phase D：订阅与学校授权双轨

### 交付

- 个人体验：游客/学生到阈值后引导家长绑定或人工核验支付。
- 学校授权：后台展示席位数、活跃率、试用到期、班级报告。
- 真实微信商户配置就绪前，明确“演示模式/人工核验”边界，不伪装成自动支付。

### 验收

- 未配置微信时，支付页明确显示人工核验/演示订单。
- 普通学生账号不能绕过家长/教师确认直接获得完整付费权益。
- `npm run env:doctor` 能区分 offline/demo/production 三种结果。

## Phase E：运维与本地教师机稳定性

### 交付

- 更新本地启动说明，固定推荐 `localhost` 而非 `127.0.0.1`。
- 增加 `.env.offline.example` 与 `.env.production.example`。
- 增加端口占用/Origin 错误排查说明。
- CI 中把核心集成测试从 `continue-on-error` 逐步提升为硬门禁。

### 验收

- 教师机离线包能一键启动，访问地址清晰。
- 线上部署环境不会带 `ALLOW_MEMORY_FALLBACK=true`。
- Cron、AI、行情、支付失败都有用户态兜底，而不是英文白屏。

## 下一批优先级

1. 统一任务中心、市场页、策略台的“下一步任务”模型。
2. 给交易前添加轻量“行动理由”输入。
3. 移动端降低任务中心无限浮动动画。
4. 继续把支付相关文案收束到家长/教师确认模型。
5. 教师端增加课堂干预队列和出口卡。

## 本轮已落地的 Phase A/B 收口

- 主任务锦囊翻开后自动聚焦“查看任务详情”，翻回后自动聚焦“打开任务锦囊”。
- `/api/student/quests` 与 `/api/student/quests/draw` 已补试用/订阅门禁，防止过期用户继续写入任务奖励。
- 学生端横幅和确认链接按钮已从“付款链接”改为“家长确认链接/开通说明”。
- 学生可见任务奖励文案已从“抽取/典藏/差 N 张/集齐”进一步弱化为“领取/系统/已收藏/点亮”。
