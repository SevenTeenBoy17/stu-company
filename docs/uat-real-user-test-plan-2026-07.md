## 5. 合规：招募、同意、隐私、支付、激励、数据处置

> 合规基线：受试学生 12–15 岁、可能含不满 14 周岁儿童。按《个人信息保护法》第 28 条（不满 14 岁个人信息=敏感个人信息，需**单独同意**）、第 31 条（需**监护人同意**+专门处理规则）与《儿童个人信息网络保护规定》，**全员按未满 14 岁最严口径执行：无书面监护人同意 = 不入组。**

### 5.1 招募渠道与筛选

- **只经学校/教师触达**，家长（成人）代为报名；不做面向未成年人的公开招募。
- 问卷硬门槛：初一~初三在读 / 家有电脑或平板 + 稳定网络 / 监护人同意签署同意书 / 学生本人自愿（同意书含学生签名栏）。
- 报名数据最小化：只收年级、年龄段（满/不满 14 二选一）、设备类型、监护人称呼+联系方式。**不收真实全名、身份证号、照片、学号。**

### 5.2 《监护人知情同意书》十条款（缺一不可，一式两份）

1. **测试目的**：教育产品可用性测试；模拟资金，不涉及真实投资，不构成投资建议。
2. **时长与安排**（吸收缺口 #2/#12，与 §6 完全一致）：总窗口约 10 天；D0 培训半天 + 5 个现场日；**每现场日 2.5 小时，其中连续使用每 40 分钟强制休息 15 分钟**；不占用上课时间、不安排 21:30 后的任务；缺勤在补场日顺延。
3. **采集数据清单**：账号内模拟操作日志与学习进度；KeyAI 对话文本；观察记录与问卷；榜单档案（昵称/省市/学校名）。
4. **录屏与口述条款（单独勾选，可单独拒绝）**：屏幕录制+出声思考录音，仅用于产品改进；不拍摄人脸；保存期限见条款 6。
5. **随时退出**：无需理由，退出不影响在校评价，可要求删除全部数据。
6. **保存期限与删除承诺**：操作日志与问卷 90 天内删除或匿名化；录屏/录音报告定稿后 30 天内删除；附删除申请联系方式。
7. **数据不外流**：不出售、不共享第三方、不用于广告；对外汇报只用脱敏数据。
8. **付费声明**：学生账号不进行任何真实支付（§5.4）。
9. **法律依据**：明示 PIPL 第 28/31 条单独同意。
10. **双签名**：监护人 + 学生本人 + 日期。另如实告知：隐身设置管辖跨校公开榜；班级内赛季记分牌为同班可见。

执行规则：同意书未回收→不发账号；口头同意=未同意；D0 现场核验回收（§6.1）。

### 5.3 榜单隐私红线（沿用产品内已有双闸门并加严）

| 红线 | 执行 | 验证 |
| --- | --- | --- |
| 昵称禁真名/拼音缩写/学号 | D0 培训统一起 `UAT-` 前缀昵称；onboarding 时观察员当场查验 | D1 结束导出全部 alias 人工复核 |
| 默认建议 `school_only` | 培训话术："先选仅校内，想公开随时可改"（S 表中 public 者凭同意书条款执行） | 抽 3 账号跨校验证不可见 |
| 未勾监护人同意=不上榜 | 沿用产品 onboarding 闸门（未勾选可保存但不进公开榜） | 四作用域均不可见 |
| 真名/班级不进公开榜 | 产品端榜单仅昵称+学习点 | 用异校账号请求 board 接口，响应无真名/班级字段 |

### 5.4 支付测试安全（总红线：未成年人零真实资金操作）

- 学生账号全程停留 trial/degraded/expired 免费态——降级体验本身是测试项。
- 付费链路只由**成人**验证，两方案：
  - **方案 B（默认）**：成人买家上传带"测试件"水印的模拟凭证 → 管理员走确认/驳回两分支，零资金流动。
  - **方案 A（每链路 1–2 次）**：成人 0.01 元真实转账至项目组测试收款码 → 传凭证 → 管理员确认 → 验证订阅翻转；结束当日原路退还并记录。
- 附加：测试机不登录任何真实微信支付；收款码测试后立即撤配；凭证截图仅管理员可见、按 §5.6 删除、永不进汇报材料；学生误触购买入口只会到"上传凭证"页（无真实支付可能），该情形记一条 UX 发现。

### 5.5 激励（合规非现金，与产品"去射幸"原则一致）

| 对象 | 激励 | 条件 |
| --- | --- | --- |
| 学生（完成 ≥10h + 问卷） | 「财商小测评官」纸质证书 + 文具（≤30 元）+ 产品内测试纪念装饰卡（确定性发放） | 完成即得，人人有份 |
| 学生（中途退出） | 按时长发参与证书，不区别对待 | 兑现"退出无不利"承诺 |
| 教师 | 感谢函 + 一学期班级 Premium | 完成教师脚本 |
| 家长 | 家庭 Premium 延长 3 个月 | 完成家长脚本+周报回访 |

**不设榜单名次奖**——奖品挂钩排名会诱导刷分、污染数据、违背"排纪律不排运气"的教学定位。

### 8.4 跨角色联动用例（每条 UAT 期间至少完整跑 2 次）

**L3 买家传凭证 → 管理员确认 → 订阅变化**：付款页三要素与配置逐字一致 → 提交后买家可见"待确认"中间态（不得显示已开通）→ 管理员确认 → 订阅 active + 记录"提交→到账"总时长（付费体验核心指标）；家长购 premium 时追加 L2 验证家庭同步。

**通过标准**：三链路各 2 次执行双端证据齐备、真实用户无需技术介入即可独立走通、同步延迟 ≤1 次手动刷新；任何"操作成功但对端不可见"记 P1。

**证据采集脚本（吸收缺口 #10：不收集未成年人会话凭证）**——由**运维本人**用项目组持有的成年观察账号（每地区 1 个，D-2 预置）登录后执行；学生侧 `/me` 证据一律为学生自查截图上交：

```powershell
$ts = Get-Date -Format "yyyyMMdd-HHmm"
New-Item -ItemType Directory -Force evidence | Out-Null
foreach ($p in "weekly","monthly","season") {
  foreach ($s in "school","city","province","nation") {
    curl.exe -s -b ops-observer-cookies.txt "https://<生产域名>/api/leaderboard/board?period=$p&scope=$s&pageSize=50" `
      -o "evidence/board_${p}_${s}_$ts.json"
  }
}
```

**刷新专项判定（任一不满足记 P1）**：周一 08:00 后 weekly 键严格递增且旧数据不带入新周榜；刷新瞬间 monthly/season/个人段位零变化（三份 diff 为空）；hidden/school_only 语义在新周不变（V1–V3 复验）；新周全部新局共享同一新种子且异于上周。
