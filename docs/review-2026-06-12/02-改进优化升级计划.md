# Brown Zone 改进优化升级计划

> 配套文档：问题编号引用自 `01-问题汇总.md`。
> 原则：**安全 > 教学诚信 > 性能稳定 > 体验一致 > 边角打磨**。每个 Sprint 有明确的"完成定义（DoD）"与回归测试要求；不允许在没有对应测试的情况下关闭 P0/P1。

## Sprint 1 — 教学诚信重设计（反刷分）

> 这是产品层面最重要的一轮：现状下"不参与 / 死押确定性资产 / 无限杠杆 / 纯点击刷课"的学生排名高于真正动脑的同学，与财商教育目标相悖。DOM-01/02/03 同源，必须**整体重设计**而非逐条打补丁。

### S1-1 资产与杠杆模型重构（DOM-01）
**改动文件**：`src/lib/simulation.ts`、`src/lib/market-data.ts`
- 房产/创业估值接入 `seed`/`eventTimeline`，引入与市场相关的波动与下行（不再是确定性单调升值曲线）。
- `loan` 增加额度上限：`maxDebt = round(netWorth * LEVERAGE_CAP)` 或起始本金固定倍数；超额拒绝并给中文提示。
- 债务利率 ≥ 任一资产的确定性收益上限，杜绝"借钱买确定升值资产"的无风险套利。

### S1-2 战力评分模型修正（DOM-02 / DOM-05 / DOM-09）
**改动文件**：`src/lib/leaderboard/power-score.ts`、`src/lib/leaderboard/run-power.ts`、`src/lib/leaderboard/service.ts`
- `riskAdjReturn`：零收益/零波动给低分（如 0.2），不再白送 0.5；样本 <3 个收益时设中性而非允许 `/epsilon` 饱和。
- `drawdown` 满分要求曾承担过敞口（从未投资不应拿满）。
- 纪律分（`simulation.ts:151`）去掉"囤现金 +6"或改为"该出手时出手"的主动性度量；引入杠杆率/集中度惩罚项。
- power 计算要求"最少有效回合数 / 最少有效交易数"，否则不计入榜单。
- `powerFormula()` 透明面板公开归一化常量与各分项保底/封顶语义（对学生诚实）。

### S1-3 种子校验与完赛要求（DOM-03）
**改动文件**：`src/lib/leaderboard/service.ts`、`src/lib/db/repo.ts`、`drizzle/`（新迁移）
- `leaderboard_snapshots` 增列 `seed`；写快照时持久化 `run.seed`。
- 周榜读取校验 `seed === seasonSeed(periodKey)`；只取"当周种子且完成 ≥N 回合"的快照。
- 未完赛 run 不计入或显著打折；replay（`replayRunForUser`）产生的 off-season 种子 run 不进当周公平榜。

### S1-4 学习分验证（DOM-04）
**改动文件**：`src/app/api/learn/complete/route.ts`、`src/lib/db/repo.ts`、`learning_progress` 相关
- `markModuleComplete` 绑定最小完成性证明：每模块配 2–3 题小测，通过才记完成；或记录最短交互时长 + 关键交互事件。
- 过渡期可先降低纯打卡的 power 权重，待小测上线再恢复。

**Sprint 1 DoD**：
- 新增 power-score 单测：do-nothing 基线分显著低于"认真操作"样本；房产平推/无限杠杆套路不再进 tier5；≤2 快照不饱和。
- 新增 service 单测：replay/未完赛/跨周 run 不污染当周榜。
- 透明面板展示的常量与实际计算一致（可加一致性测试）。

## Sprint 2 — 性能与一致性

### S2-1 沙盘状态按班级过滤（BE-01）
`repo.ts` 的 `getSimulationStateForUser` 把 `selectAllUsers/selectAllRuns` 换成 `where(eq(classroomId, cid))`，复用现有索引。**预期**：单次读取数据量从"全库"降到"单班"，跨区传输时间大幅下降。

### S2-2 推进回合长链路拆解（BE-02）
- `recomputePowerForUser` 改异步（Next 16 `after()` 或轻量队列），不阻塞 `/advance-round` 响应。
- advance 与 state 合并同事务、复用已读 run。
- 三 period upsert 合并单事务。
**预期**：单次推进 DB 往返从 ~25 降到个位数，超时率显著下降。

### S2-3 榜单 SQL 分页（BE-03 / BE-04）
- 迁移：`leaderboard_snapshots` 反规范化 `schoolId/cityCode/provinceCode/visibility`（与 S1-3 的 seed 列同一迁移落地）。
- 读榜改 `RANK() OVER (PARTITION BY scope ORDER BY power DESC) + LIMIT/OFFSET`；viewerRank 用 `COUNT(*) WHERE power > ?` 单查询。
- recompute cron 分批 + 批内并发 + 续跑；或 `INSERT … SELECT` 批量重算。

### S2-4 写竞争与唯一约束（BE-05 / BE-06）
- `selectRunForUser` 在写路径加 `.for("update")`，或 `scenario_runs` 加 `version` 乐观锁列。
- `scenario_runs.user_id` 加唯一约束（确认产品为单 run），`onConflictDoNothing` target 改 `user_id`。

### S2-5 写回退漏洞与时区（BE-07 / BE-08）
- `"appendAiMessages"` 加入 `WRITE_FNS` + 补审计测试。
- 统一 `Asia/Shanghai` 偏移计算所有 period/season key（`periods.ts` + `season.ts` 同一基准）；**注意这会改变现有桶归属，需在版本发布说明里告知并选低峰时段切换**。

**Sprint 2 DoD**：新增/更新 leaderboard 与 repo 测试全绿；本地 + 预览压测确认推进回合 P95 延迟下降；时区切换前后桶归属变化已记录。
