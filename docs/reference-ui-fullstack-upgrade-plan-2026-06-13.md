# Brown Zone 多元投资理财教学化全栈升级计划

日期：2026-06-13  
配套文档：`docs/reference-ui-deep-review-2026-06-13.md`  
目标用户：14-19 岁学生、教师、家长、学校管理员  
设计原则：现实理财结构、教育模拟边界、游戏化留存、AI 教练复盘、未成年人友好

## 1. 产品目标

把 Brown Zone 从“经济沙盘和多个功能页”升级为“青少年多元投资理财学习操作系统”。产品不复制支付宝或券商 App 的真实交易逻辑，而是吸收其成熟的信息组织方式，转译成适合学生的学习流程。

核心目标：

- 现实感：覆盖行情、自选、基金/ETF、持仓、定投、目标账户、现金流、信用、保险、风险保护、机会识别。
- 教学性：每个指标解释一个概念，每个操作沉淀为复盘证据。
- 可玩性：任务、赛季、徽章、班级共识、观察单和挑战让学生愿意持续回来。
- 安全性：所有行为都是模拟和学习，不承诺收益，不做开户导流，不给保证式买卖结论。
- 商业化：免费体验展示价值，订阅解锁更完整 AI 复盘、赛季挑战、家长/教师报告和历史导出。

## 2. 信息架构重组

### 2.1 四大一级域

| 一级域 | 路由 | 学生问题 | 核心模块 |
| --- | --- | --- | --- |
| 首页 | `/student` | 今天我该先做什么？ | 今日任务、市场温度、财富快照、服务台、赛季 |
| 行情 | `/student/market` | 市场正在发生什么？ | 指数、观察池、板块热度、同学热度、AI 解读 |
| 机会 | `/student/opportunity` | 哪些主题值得观察？ | 长期机会、阶段热点、观察单、证据链训练 |
| 持有 | `/student/wealth` | 我的配置健康吗？ | 净值、收益拆解、持仓、目标、保护伞、复盘 |

### 2.2 二级服务分组

| 分组 | 页面 | 作用 |
| --- | --- | --- |
| 策略行动 | `/student`、`/student/market`、`/student/opportunity` | 训练观察、判断和模拟决策 |
| 资产成长 | `/student/wealth`、`/student/fund-lab`、`/student/auto-invest` | 训练长期配置、分散和定投 |
| 生活理财 | `/student/life`、`/student/goal-accounts`、`/student/protection`、`/student/credit` | 把投资和生活目标、风险缓冲连接 |
| 学习留存 | `/student/quests`、`/student/history`、`/student/rank`、课程模块 | 形成任务、复盘、排行和教师评价 |

## 3. 新增或强化的界面方案

### 3.1 学生首页服务台增强

目标：像成熟理财 App 的首页一样给学生“今日入口”，但用教育语言承接。

前端要求：

- 首屏显示市场温度、财富快照、今日任务、赛季进度。
- 服务宫格只展示高频入口，其余收纳到“更多服务”。
- 每个入口显示“你会学到什么”，降低认知负荷。
- 移动端采用短卡片和横向滑动，不堆长报表。

后端要求：

- `buildStudentHomeHubPayload(run)` 聚合任务、赛季、市场温度、财富状态。
- 今日任务来自真实行为，而不是静态按钮。
- 今日任务完成状态根据 `actionLog`、持仓、目标账户、保护伞记录推导。

### 3.2 财富持有总入口

目标：承接支付宝基金持有页心智，形成学生版“我的持有”。

前端结构：

1. 当前净值、本回合变化、风险/纪律评分。
2. 收益拆解：持仓收益、储蓄收益、房产/创业变化、债务成本。
3. 持有分类：现金、储蓄、股票/ETF/债券/商品/外汇、房产、创业、债务。
4. 快捷分析：持仓分析、交易记录、投资计划、历史复盘。
5. 目标账户和保护伞状态。
6. Mr.Brown 三段式建议。

后端结构：

- `src/lib/allocation.ts` 继续作为财富聚合层。
- `src/lib/wealth-review.ts` 负责复盘记录和建议。
- `GET/POST /api/student/wealth-summary` 提供总览和复盘提交。
- 写入 `actionLog`，短期不新增表。

### 3.3 市场观察台升级

目标：从行情展示升级成教学化观察台。

前端结构：

- 顶部市场温度卡：热、温、冷，并解释“情绪不等于买卖信号”。
- 观察池：加入、移除、记录观察理由。
- 主卡：价格、趋势、K 线或简线、教学评分。
- 六维雷达：价格动量、波动风险、行业热度、AI 相关度、趋势完整度、观察优先级。
- 板块热度：展示来源和反思提示。
- 同学热度：只显示脱敏聚合，不展示个人隐私。

后端结构：

- `GET /api/market/board?symbol=MU` 保持完整 payload。
- `GET/POST /api/student/watchlist` 管理观察池。
- `GET /api/market/peer-heat` 读取班级聚合热度。
- AllTick 不可用时回退到“教学观察池模式”。

### 3.4 机会训练场

目标：把“长期机会、阶段热点、资金流、板块热度”转成学生观察训练。

前端流程：

1. 选择机会主题。
2. 选择观察理由：政策、资金、估值、技术、情绪、学习价值。
3. 写下证据链和反方风险。
4. 选择下一步验证动作。
5. 提交后进入历史复盘和任务进度。

后端结构：

- `src/lib/opportunity.ts` 生成机会卡和评分。
- `POST /api/student/opportunity` 写入 `actionLog`。
- AI 可点评观察质量，但不输出买卖建议。

### 3.5 基金/ETF 实验室

目标：把基金市场、自选、对比和持有转成“分散投资实验”。

前端结构：

- 虚拟基金池：宽基成长、债券锚、黄金防守、全球科技、AI 主题等。
- 组合方案：防守、均衡、成长。
- 指标：预期波动、最大回撤、分散度、费用率、跟踪误差。
- 操作：预览组合、记录理由、提交实验。

后端结构：

- `GET /api/student/fund-lab?plan=&amount=` 支持即时预览。
- `POST /api/student/fund-lab` 支持记录实验。
- 历史复盘可以召回基金实验记录。

### 3.6 目标账户与保护伞

目标：让学生明白理财服务生活目标，而不是只追收益。

目标账户：

- 应急金、电脑、研学、创业启动金。
- 每个目标展示进度、缺口、每回合计划。

保护伞：

- 应急金、保险、债务、收入、分散、复盘纪律。
- 突发事件压力测试：手机损坏、家庭支出、市场回撤、创业现金流紧张。

## 4. 后端开发计划

### 4.1 短期策略：actionLog-first

短期不新增数据库表，优先把所有学习行为写入 `scenario_runs.actionLog`。

| 行为 | 建议 type | 建议 meta |
| --- | --- | --- |
| 加入观察池 | `watchlist` | `kind: "watchlist_action"` |
| 机会观察单 | `opportunity` | `kind: "opportunity_note"` |
| 基金实验 | `fund_lab` | `kind: "fund_lab_action"` |
| 目标账户 | `goal_account` | `kind: "goal_account_action"` |
| 保护伞预案 | `protection` | `kind: "protection_review"` |
| 财富持有复盘 | `wealth_review` | `kind: "wealth_review"` |
| 赛季奖励 | `quest` | `kind: "season_reward_claim"` |

优点：

- 离线演示和本地运行更稳定。
- 不立即扩大数据库迁移和 RLS 面。
- 历史复盘、任务中心、AI、教师/家长报告都能统一读取。

### 4.2 中期策略：独立表正式化

当玩法验证稳定后，再拆表：

| 表 | 用途 |
| --- | --- |
| `student_watchlists` | 学生观察池 |
| `opportunity_notes` | 机会观察单 |
| `fund_lab_records` | 基金实验记录 |
| `goal_accounts` | 目标账户 |
| `protection_reviews` | 风险保护伞预案 |
| `season_rewards` | 赛季奖励 |

每张表必须具备：

- `user_id` 外键。
- `classroom_id` 或可推导班级。
- `created_at`。
- RLS 策略。
- 教师和家长可见范围约束。

### 4.3 API 规范

所有新增或打磨 API 必须满足：

- 请求体和 query 使用 zod 校验。
- 错误统一为 `{ error, message }`。
- 未登录返回 401，角色不符返回 403。
- 数据库失败返回中文 503，不让页面白屏。
- AI 调用只能通过 `src/lib/ai.ts`。
- 行情接口必须字段级 fallback。
- 写操作要防重复提交，必要时使用 action hash 或短时间幂等保护。

## 5. 前端设计计划

### 5.1 视觉原则

- 保持 Brown Zone 深色、暖琥珀、浅色内容区的品牌气质。
- 多用横向矩形信息卡，少用过窄竖卡。
- 移动端首屏优先摘要，不强行展示大图表。
- 图表只服务概念解释，不制造专业交易终端压迫感。
- 金额、收益、风险、任务进度有清晰层级。

### 5.2 交互状态

所有可点击组件必须覆盖：

- 默认状态。
- hover / focus。
- loading。
- disabled。
- 成功反馈。
- 错误反馈。
- 空状态。

### 5.3 动效原则

- GSAP 用于页面入场、关键卡片 reveal、数值变化、任务完成反馈。
- 不让动效干扰阅读。
- 遵守 `prefers-reduced-motion`。
- 奖励动画只在完成任务、领取徽章时出现。

## 6. AI 导师计划

### 6.1 统一输出协议

所有学生端 AI 反馈采用三段式：

```text
我观察到：
...

这说明：
...

下一步模拟动作：
...
```

### 6.2 页面上下文

| 页面 | AI 关注点 |
| --- | --- |
| 首页 | 今天最值得完成的学习动作 |
| 市场 | 解释行情指标和观察池 |
| 机会 | 评价证据链质量 |
| 财富 | 解释配置、现金垫和持有节奏 |
| 基金实验 | 对比波动、回撤、费用和分散 |
| 保护伞 | 解释风险缓冲是否足够 |
| 历史复盘 | 总结行为模式和下一步改进 |

### 6.3 认知负荷控制

面向 14-19 岁学生，每次 AI 反馈最多引入一个新概念，例如：

- 最大回撤。
- 分散投资。
- 现金安全垫。
- 机会成本。
- 资金流。
- 风险收益比。
- 行为偏差。

## 7. 游戏化与留存设计

### 7.1 每日循环

```text
看市场温度 -> 读今日必看 -> 写观察单或做模拟配置 -> 推进回合 -> 看 AI 复盘 -> 领取学习徽章
```

### 7.2 每周循环

```text
完成赛季主题 -> 班级共识比较 -> 历史复盘 -> 教师点评 -> 家长报告摘要
```

### 7.3 排行榜重构建议

排行不只看净值，建议拆成：

- 决策纪律。
- 分散配置。
- 学习完成。
- 复盘质量。
- 风险保护。
- 净值表现。

这样可以避免学生只追高风险收益。

## 8. 商业模式处理

### 8.1 免费体验

免费用户应可体验：

- 首页服务台。
- 部分市场观察。
- 一次机会观察单。
- 一次基金实验。
- 基础历史复盘。

### 8.2 个人订阅

订阅用户可解锁：

- 完整 AI 个性化复盘。
- 更多赛季主题。
- 历史报告导出。
- 家长成长报告。
- 更多基金/目标/保护伞场景。

订阅不能让学生在排名中获得不公平收益，也不能直接提高模拟收益。

### 8.3 学校授权

学校授权重点：

- 批量账号。
- 班级任务。
- 教师点评。
- 班级看板。
- 学习报告。
- 校际挑战赛。

学校授权比个人订阅更适合长期商业化，应在超级管理员后台里保留运营视角。

## 9. 测试计划

### 9.1 单元测试

覆盖：

- 市场温度计算。
- 观察池加入/移除。
- 机会观察评分。
- 基金组合风险、回撤、分散度。
- 目标账户进度。
- 保护伞压力测试。
- 财富复盘生成。
- 赛季任务完成状态。

### 9.2 API 测试

覆盖：

- 未登录访问返回 401。
- 非学生访问学生 API 返回 403。
- 请求体错误返回 400。
- DB 不可用返回中文 503。
- 重复提交不重复发奖励。
- AI 不可用时返回本地兜底。

### 9.3 E2E 测试

核心链路：

1. 学生登录。
2. 进入首页服务台。
3. 查看市场温度。
4. 加入观察池并写理由。
5. 提交机会观察单。
6. 做一次基金实验。
7. 设置目标账户或保护伞。
8. 推进一回合。
9. 历史复盘显示上述动作。
10. AI 输出教育化建议。

### 9.4 视觉测试

视口：

- `390x844`
- `768x1024`
- `1440x980`

检查项：

- 无横向滚动。
- 数字不溢出卡片。
- 移动端首屏有明确主任务。
- 按钮状态完整。
- 长图表不挤压移动端。
- 上涨红、下跌绿保持一致。
- AI 悬浮按钮不遮挡关键操作。

## 10. 阶段排期

| 里程碑 | 估计时间 | 交付 |
| --- | --- | --- |
| M1 | 2-3 天 | 学生首页服务台和四大一级域收敛 |
| M2 | 3-4 天 | 财富页升级为持有总入口 |
| M3 | 3-5 天 | 市场页教学链和观察池闭环 |
| M4 | 3-5 天 | 机会训练场接入历史、任务和 AI |
| M5 | 3-5 天 | 基金/ETF 实验室预览、记录和复盘 |
| M6 | 3-5 天 | 目标账户、保护伞、生活理财联动 |
| M7 | 4-6 天 | 赛季、任务、教师/家长报告联动 |
| M8 | 2-3 天 | 全链路 E2E、视觉审阅、上线准备 |

## 11. 推荐实施顺序

不要再做大范围重构。建议按“一次只打通一个闭环”的方式推进：

1. 清理学生端首页和服务入口。
2. 完成财富持有总入口。
3. 完成市场观察池和 AI 解读闭环。
4. 完成机会观察单进入历史复盘。
5. 完成基金实验进入历史复盘。
6. 完成目标账户和保护伞进入财富地图。
7. 补教师、家长、管理员报告视角。

## 12. 终审结论

本计划可执行，且技术风险可控。当前项目已经有足够多的模块基础，下一阶段的关键不是继续堆新页面，而是把“行为记录、复盘、任务、AI、报告”串成闭环。参考图里的成人金融功能应当只借鉴结构，不借鉴高压营销和真实交易导向。

最终建议：优先执行 M1 到 M5，完成学生端核心体验闭环后，再扩展更复杂的税费、保险、相关性、费率、家庭预算等进阶内容。
