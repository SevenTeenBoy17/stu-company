# 监工B使用说明

## 目的

`monitor_b` 是 Brown Zone 项目专用的只读监督 agent，用来检查开发是否严格遵守：

- `03-多元理财功能升级与全栈开发计划.md`
- `03b-参考图实景分析与映射.md`
- `05-CODEX执行手册.md`
- `AGENTS.md`
- `CODEX-WORKFLOW.md`

它不写代码，只负责阶段门禁、红线检查、偏离处理建议和最终审阅结论。

## 推荐调用时机

1. 每个 Sprint 或功能 Phase 开始前，让 `monitor_b` 做前置闸门审查。
2. 每个实现 agent 完成后，让 `monitor_b` 审查 diff、阶段顺序、禁区越界和验证证据。
3. 任何涉及迁移、安全、JWT、评分模型、支付、AI、时区、排行榜、奖励机制的改动前，先让 `monitor_b` 判断是否需要 stop-gate。
4. 发布前，让 `monitor_b` 执行 F.4 风格终审。

## 示例提示词

### 阶段开始前

```text
Have monitor_b run a stage-entry audit for Phase 1 C-1/C-2/C-8.
Check whether S0, S1, FE-01 and F.0 prerequisites have evidence.
Do not edit files. Output APPROVE / REQUEST_CHANGES / NEEDS_DISCUSSION.
```

### 实现完成后

```text
Have monitor_b audit the current diff against docs/review-2026-06-12/03-多元理财功能升级与全栈开发计划.md and 05-CODEX执行手册.md.
Focus on stage order, forbidden scope, AI gateway, decorative-only rewards, privacy, tests, and UI three-state requirements.
Do not edit files. Output findings with severity and next required fix.
```

### 偏离修复后

```text
Have monitor_b re-audit only the files changed for the previous REQUEST_CHANGES.
Confirm whether the blocker is fixed and whether we can proceed to the next gate.
Do not widen scope.
```

## 与其他 agent 的配合

| 任务类型 | 主责 agent | 建议审阅 agent | 监工B关注点 |
| --- | --- | --- | --- |
| 数据库/迁移 | `db_architect` | `engineering-database-optimizer` | 禁用 `drizzle-kit push`、迁移 stop-gate、FK/RLS/回滚 |
| API/Auth | `api_wirer` / `engineering-backend-architect` | `engineering-code-reviewer` | zod、origin、rate limit、错误形状、权限过滤 |
| UI/前端 | `ui_implementer` / `engineering-frontend-developer` | `design-ui-designer` / `testing-accessibility-auditor` | 三态、可访问性、响应式、无乱码、无竖排导航 |
| 模拟/评分 | `finance_event_simulator` | `reviewer` | 反作弊、确定性、奖励仅装饰、排行榜公平 |
| AI | `behavior_ai_analyst` | `engineering-code-reviewer` | 所有 AI 调用只走 `src/lib/ai.ts`、兜底、教育边界 |
| 支付/订阅 | `monetization_wechat_engineer` | `compliance-auditor` | 未成年人付费边界、幂等、密钥、回调安全 |
| 测试/发布 | `qa_engineer` | `testing-reality-checker` | 质量门、Playwright、手动验收、发布阻断项 |

## 监工B不会做什么

- 不直接修改代码。
- 不替代实现 agent。
- 不替代产品决策。
- 不允许用“看起来没问题”代替命令输出、截图、测试或 grep 证据。
- 不允许在 S0/S1 未达标时推进多元理财功能升级。

## 快速确认

运行 `/agents` 时应能看到：

```text
monitor_b
```

如果看不到，检查文件是否存在：

```powershell
Get-ChildItem .\.codex\agents\monitor_b.toml
```
