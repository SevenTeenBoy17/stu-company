> 目标：参考支付宝（蚂蚁财富）「理财」页的产品形态，把 Brown Zone 从"12 回合炒股沙盘"升级为"贴近现实的多元理财教学平台"，同时强化游戏化、趣味化、竞争力与教学性，让 14–19 岁学生**玩中学、学中玩**，提升使用率、订阅续费率与留存率。

## Part A — 支付宝（蚂蚁财富）「理财」页 UI 拆解

### A.1 页面信息架构（自上而下的模块与层级）

```
┌─ 顶部资产总览区 ────────────────────────────┐
│  总资产 ¥ ____   [👁 隐藏金额]                │
│  昨日收益 +__   累计收益 +__   持有金额 __     │  ← 收益拆解
│  └ 资产走势小图（近30日曲线）                  │
├─ 快捷功能金刚区（图标网格）───────────────────┤
│  [转入] [转出] [收益] [基金] [黄金] [更多]     │
├─ 稳健理财 / 活钱管理 ────────────────────────┤
│  余额宝类：七日年化 __%  万份收益 __  随时取    │  ← 低风险、流动性
├─ 进取理财 / 持仓 ───────────────────────────┤
│  基金/股票卡片：净值 __  日涨跌 +__%  [加仓][定投]│  ← 中高风险
├─ 智能投顾 / 帮你投 ──────────────────────────┤
│  风险测评 → 资产配置建议（环形图：股X% 债Y% 现Z%）│
├─ 定投专区 ──────────────────────────────────┤
│  定投计划：每周/每月自动买入 __  已坚持 __ 期     │  ← 纪律性投资
├─ 内容/教育流（财富号·直播·资讯）──────────────┤
│  图文/视频卡片：理财知识、市场解读              │
└─ 活动/福利/任务（红包、积分、签到）────────────┘
```

### A.2 核心 UI 元素与它们之间的关系

| 元素 | 作用 | 与其它元素的关系 |
|---|---|---|
| **总资产卡片** | 单一焦点，建立"我的钱"心智 | 点击下钻 → 收益拆解 → 持仓明细；隐藏金额 toggle 保护隐私 |
| **收益拆解**（昨日/累计/持有） | 把"涨跌"翻译成"赚了多少钱" | 喂养资产走势曲线；是用户每日打开 App 的核心动机 |
| **金刚区快捷入口** | 降低高频操作的路径深度 | 每个图标对应一个资产类别/动作页 |
| **稳健 vs 进取分区** | 用"风险高低"组织产品，做风险教育 | 与智能投顾的"资产配置环形图"呼应：环形图各扇区即这两区的产品 |
| **七日年化 / 万份收益** | 货币基金的标准话术 | 教学概念锚点（复利、流动性、低波动） |
| **资产配置环形图** | 可视化"鸡蛋不放一个篮子" | 由风险测评驱动；与持仓实际占比对比，产生"再平衡"提示 |
| **定投计划** | 把"择时"替换成"纪律" | 与日历/坚持期数挂钩，是留存与教学的双料抓手 |
| **风险测评问卷** | 给用户贴风险标签（保守/稳健/进取） | 决定智能投顾建议；决定默认资产配置 |
| **内容流（财富号）** | 用知识填满"看完收益之后"的注意力 | 与持仓联动（你持有的标的的解读）；是低成本的高频回访理由 |
| **活动/签到/积分** | 行为激励，制造回访习惯 | 与收益日历叠加形成 streak |

### A.3 设计带给我们的三条可迁移原则

1. **以"总资产 + 收益拆解"为单一焦点**：把抽象的涨跌翻译成"今天赚/亏了多少钱"，这是每日回访的引擎。
2. **用"风险高低"而非"资产种类"组织信息**：稳健区/进取区 + 环形配置图，天然就是风险分散教育。
3. **把"择时"系统性替换为"纪律"**：定投、再平衡、风险测评——这些正是财商教育最该传递的、也是最适合做成游戏化坚持机制的。

## Part B — 迁移到 Brown Zone 的设计约束（重要）

在新增功能前，必须先尊重两条现有约束，避免"违和"与"反教学"：

1. **必须与 `02-改进优化升级计划.md` 的 Sprint 1（反刷分重设计）合并实施。** 现状下房产/创业是确定性升值、杠杆无上限、"不操作"基线高达 1110 战力（见 `01-问题汇总.md` DOM-01/02/03）。若在未修复时引入"定投/低风险稳健产品"，只会**新增更多无风险刷分路径**。新资产类别（黄金/债券/指数基金）必须接入 `seed`/`eventTimeline` 的真实波动。
2. **现有数据模型已是多资产**：`ScenarioRun` 已有 `cash / savings / debt / propertyUnits / ventureStake / holdings[]`，动作已有 `trade / bank / property / venture`（`src/lib/simulation.ts`）。本计划**不是推倒重来**，而是：① 在现有资产上叠加"理财页式"的呈现与教学层；② 新增 2–3 类资产与 1 个核心机制（定投）；③ 增加内容/任务/风险测评三个留存层。
3. **颜色约定不变**：红=涨/正、绿=跌/负（中国市场惯例，`globals.css` 令牌 `--up-*`/`--down-*`），但需先修复 FE-06（硬编码色漂移）。

## Part C — 新增界面与功能（逐项全栈开发说明）

> 每项含：教学目标 / 游戏化设计 / UI 结构 / 前端实现 / 后端·API / 数据库 / 验收标准。
> 路由命名遵循现有 `(platform)/student/*` 与 `src/app/api/*` 结构；组件落 `src/components/student/*`；纯逻辑落 `src/lib/*`。

### C-1 「我的财富」资产总览页（替代/升级现有 allocation panel）
**新增页**：`/student/wealth`（`(platform)/student/wealth/page.tsx`）

- **教学目标**：把涨跌翻译成"赚/亏了多少"，建立资产 = 现金+存款+持仓+房产+创业−负债 的净值心智；引入"昨日收益/累计收益"概念。
- **游戏化**：隐藏金额 👁 toggle（青少年隐私感）；净值突破历史新高时的"新高"庆祝微动效；收益日历（见 C-6）。
- **UI 结构**：顶部总资产卡（净值 + 本回合收益 + 累计收益 + 隐藏 toggle）→ 净值走势曲线（复用 `snapshots[]`）→ 大类资产配置环形图（C-2）→ 稳健区/进取区分组列表 → 入口卡（定投 C-3、风险测评 C-4、学习 C-5）。
- **前端实现**：
  - Server Component 拉首屏数据（净值、snapshots、holdings），客户端只在交互（toggle、切曲线区间）时用 `"use client"` 子组件。
  - 净值曲线用现成图表方式（项目已有 K 线/雷达图实现可复用模式）；曲线区间切换（近 3/6/12 回合）。
  - 金额组件统一走 `MoneyText`（先修 FE-17 全角￥ + FE-06 令牌）。
  - 时间显示走修复后的 `Asia/Shanghai` 格式化（FE-05）。
- **后端·API**：复用 `getSimulationStateForUser`（**先修 BE-01 全表扫描**）；新增 `GET /api/student/wealth-summary` 返回 `{ netWorth, roundReturn, cumulativeReturn, allocation, snapshots }`（zod 校验、中文错误形状、`requireUser("student")`、按班级过滤）。
- **数据库**：无新表（数据已在 `scenario_runs`）。`netWorth` 已物化（`types.ts:160`）。
- **验收**：隐藏金额持久（localStorage）；DB 故障显示"数据暂不可用·重试"而非空白（避免 FE-02/07 同类问题）；曲线与持仓明细数值勾稽一致。

### C-2 大类资产配置环形图 + 风险分散评分
**新增组件**：`src/components/student/asset-allocation-donut.tsx`；纯逻辑 `src/lib/allocation.ts`

- **教学目标**：可视化"鸡蛋不放一个篮子"；引入"集中度风险""再平衡"概念。
- **游戏化**：分散度评分（0–100）+ 段位文案（"全押一注 ⚠️" → "攻守均衡 🛡️"）；与战力榜的"集中度惩罚项"（S1-2）共用同一指标，**前台教育 = 后台计分**，闭环。
- **UI**：环形图分区（现金/存款/债券/股票/房产/创业/黄金）+ 图例占比 + "你的配置 vs 你的风险标签建议配置"对比条 + 一句话再平衡建议。
- **前端**：纯 SVG 环形（无第三方依赖），数据来自 wealth-summary 的 `allocation`；色盲冗余（扇区加纹理/标签，呼应 FE-18）。
- **后端·逻辑**：`computeAllocation(run)` 与 `computeDiversificationScore(run)` 放 `src/lib/allocation.ts`（纯函数、可单测）；分散度指标同时被 `power-score.ts` 的集中度惩罚项消费（单一真相源）。
- **数据库**：无新表。
- **验收**：占比求和=100%；单押单一资产时分散度趋 0 且战力集中度惩罚生效（与 S1 联测）。

### C-3 「自动定投机器人」定投计划（核心新机制）⭐
**新增页/组件**：`/student/wealth` 内的定投卡 + `src/components/student/auto-invest-plan.tsx`；纯逻辑扩展 `src/lib/simulation.ts`

- **教学目标**：用"纪律性定额投资 + 平均成本法"替代"择时"，这是财商教育最该传递的反人性技能，也直接对冲现状"见好就收刷分"（DOM-03）。
- **游戏化**：拟人化"定投机器人"角色，每回合自动执行；"已坚持 N 期"徽章；坚持满 12 回合解锁成就；定投 vs 一次性梭哈的**收益对比小实验**（教学高光时刻）。
- **机制设计**：学生设置"每回合自动买入某标的 ¥X"，引擎在 `advanceSimulationRun` 时自动执行该买单（受现金约束，不足则跳过并提示）。定投行为计入纪律分（正向），**对冲**主动择时的随机性。
- **前端**：表单（选标的 + 每期金额 + 起止回合）；定投执行回执在回合推进结算里展示；动效克制（呼应 FE-13 不滥用入场动画）。
- **后端·API**：`POST /api/sim/auto-invest`（创建/修改/取消计划，`checkOrigin` + 限流 + 防双击 FE-01）；计划随 run 推进自动执行，无额外回合外请求。
- **数据库**：`scenario_runs` 增列 `autoInvestPlan jsonb`（标的/金额/区间/已执行期数），随 run 走，免新表；或独立 `auto_invest_plans` 表（若需历史）。新迁移用 `npm run db:migrate`。
- **验收**：定投在现金不足时安全跳过不报错、不产生负现金（守住金额守恒不变量）；定投 vs 梭哈对比数值正确；防双击下不重复建计划。

### C-4 风险测评问卷 + 智能投顾建议（接 AI 网关）
**新增页**：`/student/risk-profile`（`(platform)/student/risk-profile/page.tsx`）

- **教学目标**：认识自己的风险偏好（保守/稳健/进取）；理解"风险承受能力 → 资产配置"的因果。
- **游戏化**：5–7 题趣味问卷（情景化，贴近青少年："如果你的压岁钱一周跌了 20%，你会…"）；测完给"投资人格"卡片（可分享，借力 rank 的 share 能力）。
- **UI**：问卷 → 结果人格卡 + 建议环形配置 → "一键对照我的实际配置"跳 C-2。
- **前端**：问卷状态机（`"use client"`），结果卡可截图分享（复用 `leaderboard/share`）。
- **后端·API**：`POST /api/ai/risk-advice`（**只走 `src/lib/ai.ts` 网关**，AGENTS.md 红线；带限流，AI 不可用时返回规则化兜底建议——项目 AI 网关本就有 fallback 叙事）；结果落 `learning_progress` 或新 `risk_profiles`。
- **数据库**：`risk_profiles(user_id PK, risk_label, answers jsonb, updated_at)`；可见性默认私有。
- **验收**：AI 不可用时有规则兜底、不抛错；风险标签驱动 C-2 的"建议配置"对比；问卷可重测。

### C-5 财商学堂内容流（学习 = 战力，闭环）
**升级**：`/learn`（站点）+ 新增学生端「学堂」tab；强化 `learning_progress` → 战力闭环

- **教学目标**：把"看完收益之后"的注意力填进知识；学习直接转化为战力的 15%（现已存在，但 DOM-04 是零验证，本项配合修复）。
- **游戏化**：知识卡 + 课后 2–3 题小测（**这正是 DOM-04 学习分验证的载体**）；连续学习 streak；与持仓联动（"你持有的『科技成长股』为什么今天跌了？"）。
- **UI**：内容卡流（图文/短视频位）+ 小测弹窗 + 完成进度环；空态/失败态规范（修 FE-16）。
- **前端**：复用 `learn-catalog`；小测组件新增；打卡失败要有反馈（修 FE-16）。
- **后端·API**：`/api/learn/complete` 改为"通过小测才记完成"（**DOM-04 修复点**）；`/api/learn/quiz`（取题/判分，题目服务端持有正确答案，防作弊）。
- **数据库**：`learning_progress` 增 `quiz_passed`、`quiz_score`；题库 `learning_quizzes`（或随内容定义在 `content.ts`）。
- **验收**：纯点击不再记完成（DOM-04 闭合）；学习 streak 正确；小测答案不下发客户端。

### C-6 任务中心 / 成就 / 收益日历（留存三件套）
**新增页**：`/student/quests`（`(platform)/student/quests/page.tsx`）

- **教学目标**：把分散的好习惯（分散投资、坚持定投、完成学习、不过度交易）显性化为可达成目标。
- **游戏化**：每日/每周任务（"今天完成 1 节学堂""本周保持分散度 ≥60"）；成就墙；收益日历签到 streak（呼应支付宝签到）；任务奖励仅给"装饰性"激励（头像框/称号），**不直接给战力**，避免又开一条刷分路（守住 S1 诚信）。
- **UI**：任务列表（进行中/已完成）+ 成就墙 + 日历 streak。
- **前端**：任务进度从既有行为派生（学习/交易/分散度），尽量"读"而非"写"，降低复杂度。
- **后端·API**：`GET /api/student/quests`（聚合派生 + 已领取状态）；`POST /api/student/quests/claim`（领装饰奖励，`checkOrigin` + 幂等）。
- **数据库**：`quest_progress(user_id, quest_key, claimed_at)`、`achievements(user_id, badge_key, earned_at)`。
- **验收**：任务进度与真实行为一致；奖励不污染战力；签到 streak 跨时区按 `Asia/Shanghai`（与 BE-08 同步）。

### C-7 新增资产类别：黄金（避险）+ 指数基金（分散）
**扩展**：`src/lib/market-data.ts`、`src/lib/simulation.ts`、`MarketWatchlistSymbol`

- **教学目标**：避险资产（黄金，与股市低/负相关）+ 一篮子分散（指数基金，单只 ≠ 集中度）。让"资产配置环形图"真正有 ≥4 个有意义的大类。
- **机制**：黄金/基金接入 `seed`/`eventTimeline` 的真实波动（**必须，否则又是无风险刷分**）；黄金在"利空/避险"事件下逆势，强化分散价值的体感。
- **前端**：市场页（`student-market-board`）观察池纳入新标的；圆环中心数量改 `payload.watchlist.length`（修 FE-20）。
- **后端·逻辑**：`market-data.ts` 增资产定义与事件影响方向（呼应 DOM-07 叙事一致性）；`simulation.ts` trade 动作天然支持新 holdings，无需新动作类型。
- **数据库**：无新表（holdings 是 jsonb）。
- **验收**：黄金与股票在避险事件下走势相反、可被分散度评分捕捉；新增标的不破坏金额守恒与确定性测试。

## Part D — 留存 / 续费杠杆（显式对齐商业目标）

| 杠杆 | 对应功能 | 作用机制 |
|---|---|---|
| **每日回访动机** | C-1 收益拆解 + C-6 签到 streak | "今天赚/亏了多少" + 连续签到，是高频打开的引擎 |
| **坚持型留存** | C-3 定投"已坚持 N 期" + C-5 学习 streak | 沉没成本 + 进度可视，降低流失 |
| **社交/竞争拉新** | C-4 投资人格卡分享 + 现有战力榜 | 可分享卡片借同学社交链扩散（复用 `leaderboard/share`） |
| **续费价值锚点** | Premium 解锁：定投高级策略 / 学堂进阶 / 家庭报告 | 把 C-3/C-5 的进阶能力放进 Premium，让续费=解锁成长（注意：不可把"刷分能力"放进付费，否则伤公平） |
| **家长侧续费推力** | 现有周报 + C-2 分散度/C-5 学习进度纳入家长报告 | 让家长看到"孩子在学真东西"，强化付费意愿 |

> 风控红线：所有任务/成就/分享奖励只给**装饰性**激励，绝不直接发战力，否则会重新打开 DOM 系列刷分路，伤害榜单公平与教学诚信。

## Part E — 分期交付与依赖关系

```
依赖前置：必须先完成 02 计划的 Sprint 0（安全）+ Sprint 1（反刷分重设计）
          ──否则新功能会放大刷分面、且建在不稳的鉴权/性能地基上。

Phase 1（与 S1/S2 并行落地呈现层）
```
