## 0. 结论速览（TL;DR）

四项需求的底座**大部分已经存在且实现质量高**，但每一项都有"最后一公里"的缺口，直接决定转化率/续费率与"随机性/进阶性"卖点能否成立：

| 需求 | 现状 | 关键缺口（决定成败） |
|---|---|---|
| ① 微信收费 + 试用 + AI 行为评定 | ✅ 完整：状态机 / ¥15 月卡 / 微信支付 v3 / 付费墙已正确接线 | ⚠️ 学生端**没有任何应用内付费入口**（合规取舍但堵死自助转化）；"完整 AI 试用仅约 1 天"过短；premium 档位卖不出去（死路径） |
| ② 邮箱注册 + 唯一性 | ✅ 邮箱 DB 唯一约束 + 应用层校验，双保险 | ❌ **无邮箱验证** → 换邮箱即可无限刷新 3 天试用（付费漏损）；❌ 无自助找回密码 |
| ③ 游戏化 AI 教学 | ✅ 7 步叙事化新手村，单步单概念，交互+AI 兜底，符合认知负荷 | ⚠️ 一次性、无掌握度校验、无间隔复习、无术语表；预测题的"揭晓结果"**写死**、不回应学生预测 |
| ④ 随机金融事件 | ✅ 26 张事件卡 + 6 类目 + 难度分层 + 黑天鹅 + 8 类行为信号引擎 | ❌ **事件并不随机**：每回合事件与涨跌倍率全部写死，每局完全相同 → 零重玩性、零惊喜；事件文本与真实市场数学**解耦**（不影响结果）；事件无"选择/分支" |

**一句话定位建议**：当前产品="高质量但**固定剧本**的金融素养沙盘"。要做出市场差异化卖点、把留存/续费撑起来，核心是把第④项从"固定剧本"升级为"**带随机性 + 真实因果 + 玩家选择**的可重玩引擎"，并把第①②项的转化漏损口堵上。

## 1. 现状审计（逐需求 · 有据可查）

### 1.1 需求① 微信收费 / 试用 / AI 行为评定

**已实现（质量高）**
- 订阅状态机 `src/lib/billing/subscription.ts`：`trial`(满血 AI) → `trial_degraded`(基础 AI) → `expired`；付费 `active`(standard/premium)、`grace_period`、`cancelled`。试用常量 `TRIAL_FULL_DAYS=1`、`TRIAL_TOTAL_DAYS=3`（`subscription.ts:25-26`）。
- 价格正确：`STANDARD_MONTHLY = { amountFen: 1500 }` = **¥15/月**（`src/app/api/billing/prepay/route.ts:28-31`），与需求完全一致。
- 微信支付 v3 完整：native(扫码) + jsapi(微信内)、下单签名、回调验签 + AEAD-GCM 解密、mock 模式（`src/lib/billing/wechat-pay.ts`）。回调履约 `src/app/api/billing/notify/route.ts`。
- **付费墙接线正确**（这是 "AI 行为评定" 的变现核心，已严密 gating）：
  - 沙盘操作/推进回合：`canUserOperate()` 守卫 `sim/actions`、`sim/advance-round`。
  - AI 个性化评定：`canUsePersonalAiAssessment` / `aiTier` 守卫 `ai/tutor`、`ai/radar-chart`、`ai/chat`、`student/history-review`、`market/portfolio-intel`。
- 学生不能直接付款，必须经"游客升级 billing-intent"或家长/教师代付（`prepay/route.ts:62-90`）——**对"未成年人+支付"是正确的合规取舍**。

**缺陷 / 缺口**
1. **学生端零付费入口（转化致命点）**：`subscription-banner.tsx:40` 对 `isStudent` 隐藏升级 CTA，文案只说"请让家长或老师帮你解锁"。而已注册学生**无法走 guest-upgrade**（该路径 `guest-upgrade/route.ts:33` 仅限共享游客账号）。结果：一个试用到期的青少年自己想付费也**找不到任何按钮**，只能口头求家长 → 自助转化≈0。
2. **"满血 AI"仅约 1 天**：`daysRemaining > 2` 才是 full（`subscription.ts:93`），3 天试用里第 1 天满血、第 2-3 天降级为基础 AI。青少年通常 1 天内还没形成"啊哈"价值认知就被降级，**习惯养成窗口过短**（竞品基准多为 7 天，详见 §3）。
3. **premium 卖不出去**：`prepay` 只接受 `tier: z.enum(["standard"])`，但状态机/类型里 premium 存在 → **死路径**，未定义 standard vs premium 差异化权益。
4. 退款、订阅取消（`cancelled` 状态）无对应 API/UI；续费提醒（到期前 N 天）无主动触达机制。

### 1.2 需求② 邮箱注册 / 一邮箱一账号

**已实现**
- 邮箱**双保险唯一**：DB 层 `email ... .unique()`（`src/lib/db/schema.ts:18`）+ 事务内显式校验 `if (existingUser) throw "这个邮箱已经被注册过了"`（`repo.ts:762-763`）。并发安全。
- 注册校验：邮箱格式、密码 ≥8 且含字母+数字、昵称 2-16（`register/route.ts:9-18`），邮箱统一小写。登录支持邮箱 + `superadmin` 特例。均有限流。
- 新注册自动 +3 天试用（`repo.ts:792-793`）。

**缺陷 / 缺口**
1. **无邮箱验证（付费漏损 + 身份不实）**：注册不校验邮箱所有权。任何人可用一次性邮箱（10minutemail 等）无限注册 → **每个新邮箱白嫖 3 天满血/基础 AI**。"一邮箱一账号"成立，但挡不住"一人多邮箱"刷试用，直接侵蚀付费转化。
2. **无自助找回密码**：`api/auth/` 下仅 login/logout/register/register-by-invite/onboarding/guest-upgrade，**无 forgot/reset**。密码只能由管理员代重置（`admin/users/password`）。青少年忘密码=流失。
3. 无注册"年龄/家长同意"采集 → 未成年人合规（§8）缺一环。

### 1.3 需求③ 游戏化 AI 教学（认知负荷 · 青少年）

**已实现（设计扎实）**
- `src/components/student/onboarding-flow.tsx`：7 步新手村（welcome→capital→assets→first-trade→risk→event-card→ready），**单步单概念**（本金 / 风险与收益 / 下单 / 波动 / 利好利空 / 复盘），明确 CLT 取向（"每一步只学习一个概念"）。
- 交互式：本金目标滑杆、模拟下单、"先预测后揭晓"、事件卡演示；进度条；可跳过。
- AI 逐步生成讲解 + 本地脚本兜底（`onboarding-flow.tsx:111-150` ↔ `api/ai/onboarding/route.ts` ↔ `requestOnboardingNarrative`）。完成写 `onboardingCompleted`。
- 叙事人格 Mr.Brown 贯穿，"不是投资建议"安全声明到位。

**缺陷 / 缺口**
1. **预测题揭晓写死**：`risk` 步骤捕获了 `marketGuess`（up/down），但揭晓恒为 `￥120,672 / +0.56%`（`onboarding-flow.tsx:331-343`），**不回应学生的预测**。教学上"先预测—后反馈"的闭环断了，且永远"上涨"会埋下"涨才正常"的错误锚定。
2. **一次性、无掌握校验、无复习**：概念讲完即弃，无小测 gate、无术语表（glossary）可回看、无间隔复习（spaced repetition）。青少年留存的核心机制（见 §3）缺失。
3. **非自适应**：所有人同一套 7 步，不分初/进阶，无贯穿 12 回合的"技能树/知识地图"或持续故事弧。新手村之后教学叙事**断档**。

### 1.4 需求④ 随机金融事件（进阶知识与玩法）

**已实现（素材层很强）**
- 事件库 26 张（`src/lib/market-data.ts`）：6 类目 `macro/policy/sentiment/competition/black_swan/behavior`；每卡含 `signal(利好/利空/中性)`、青少年类比式 `description`、`coachingCue`、`teachingConcept`、`impactAssets`、`impactRange`、`stage(early/middle/late)`。含 2 张黑天鹅（流动性危机 / 政策大转向）。
- 难度曲线：回合主题分层（R1-4 低波动，黑天鹅置于后段），`enrichEventCard` 按类目补教学元数据。
- 行为干预引擎 `src/lib/adaptive-events.ts`：8 类行为信号（过度交易 / 报复性交易 / 羊群 / 锚定 / 不分散 / 不碰债券 / 囤现金 / 连胜），CLT 约束**每回合最多 1 警告 + 1 提示**。这是产品**最强的差异化资产**（真正在教行为金融）。

**缺陷 / 缺口（这是第④项的核心问题）**
1. ❌ **"随机"并未实现**：`marketRounds` 把每回合的 `eventId` 与 `assetMultipliers` **全部写死**（`market-data.ts:387+`），`simulation.ts` 仅 `getRound(round).eventId`（`simulation.ts:234,409,485`）。**每一局 12 回合完全相同** → 零重玩性、零惊喜、复玩学不到新东西。这与需求"增加一些随机金融事件、进一步学习进阶玩法"直接冲突。
2. ❌ **事件与市场数学解耦**：涨跌只由写死的 `assetMultipliers` 决定，事件卡的 `impactAssets/impactRange` **不参与价格计算**。事件=旁白，**机制上不影响结果**。认真的学生会发现"事件说利空但价格按剧本涨"——破坏可信度与因果学习。
3. ❌ **事件无玩家选择/分支**：事件卡是只读新闻，不是决策点。缺"事件→选择→后果"的进阶玩法承载。
4. 进阶概念（杠杆/对冲/相关性/流动性）目前只在后段事件**文字**里出现，无对应**可操作机制**（如真正的杠杆/做空/对冲工具）。

## 2. 多角色全流程内测报告（模拟内测）

> 方法：代入 5 类角色，沿真实代码路径走查"注册→教学→沙盘→事件→付费墙→续费"。标注 🟥 阻断 / 🟧 体验问题 / 🟩 正常。

**角色 A：14 岁新注册学生「林同学」**
- 🟩 注册（邮箱+密码）→ 自动 3 天试用 → 进新手村，单概念节奏舒适。
- 🟧 `risk` 预测题：选"会下跌"，揭晓仍 +0.56%，与预测无关 → 困惑"那我猜它干嘛"。
- 🟩 进入沙盘，满血 AI 雷达/复盘可用，体验好。
- 🟧 第 2 天再登录：AI 已降级"基础版"，雷达诊断变弱 → "我昨天还能用，今天怎么缩水了？"价值感受挫。
- 🟧 想再玩一局：事件与行情**与上一局一模一样** → "学完了，没新东西"，D2 流失风险高。

**角色 B：试用到期的学生**
- 🟥 Banner："试用已结束，请让家长或老师帮你解锁"，**无任何按钮/二维码/引导**（`subscription-banner.tsx:40` 学生隐藏 CTA）。学生想付也付不了 → 死胡同，转化漏到地板。

**角色 C：家长「家长账号」**
- 🟩 `/pricing` → 选择名下学生 → ¥15 下单（native 扫码 / mock）→ 回调履约 → 学生转 active。路径成立。
- 🟧 家长如何"知道"孩子试用要到期？无到期前主动通知（邮件/站内）→ 被动等孩子开口，续费靠运气。

**角色 D：教师**
- 🟩 教师可为班级学生代付（`canUserPayForTarget`）、看班级台。
- 🟧 批量为多名学生开通无"批量下单"，逐个操作成本高（B2B2C 校园场景痛点）。

**角色 E：增长黑客 / 薅羊毛用户**
- 🟥 用一次性邮箱反复注册 → **每个邮箱白嫖 3 天 AI**，无邮箱验证、无设备指纹、无支付前置 → **可无限白嫖**。对一个靠 AI 算力成本的产品=直接亏损。

**角色 F：游客（共享体验账号）**
- 🟩 demo → guest-upgrade（填邮箱注册）→ 拿 billingIntent → 可自助付 ¥15。**这是目前唯一的学生侧自助转化路径**，但仅共享游客可走，已注册学生反而走不了（见角色 B 矛盾）。

**内测结论**：核心闭环可用且稳健，但有 **3 个 🟥 阻断级问题**（学生无付费入口 / 无限刷试用 / 预测题断闭环）和多个 🟧 留存-续费体验问题，必须在交付前处理。

## 3. 竞品分析与定位（基于模型知识，未经实时核实，落地前需补数据）

> ⚠️ 本节市场数字与具体定价为模型知识（截至 2026-01）的方向性判断，非实时抓取，正式投放前请用 QuestMobile / 七麦 / App Store 榜单 / 应用商店校准。

**赛道格局（方向性，高置信）**
- 中国成人"财商教育"（启牛、微淼等）惯用**低价引流课→高价大课**漏斗，且长期处于**监管高压**（虚假宣传、诱导消费、荐股资质）。启示：**成人财商大课模式不可照搬到青少年**；"未成年人 + 模拟交易 + 收费"是敏感组合，必须以**教育/模拟**而非**投资建议/荐股**定位。
- 免费学校工具类（同花顺模拟炒股、SIFMA Stock Market Game）以"免费 + 课堂"占据校园心智 → 家长对"模拟炒股"的**默认心理价位偏免费**。
- 全球青少年金融素养 App（Zogo/Greenlight/Duolingo-for-finance 类）走**订阅/微订阅 + 游戏化**，与本产品 ¥15/月模型更接近。

**对 ¥15/月 + 1–3 天试用的判断**
- ¥15/月在中国青少年 EdTech 属"低门槛冲动价"，**定价本身有竞争力**；但本土**缺少同价位对标**，家长心智容易二分为"该免费"或"该是大课"。→ 需要用**价值叙事**（AI 个性化行为评定 + 家长成长报告）把 ¥15 锚定为"一杯奶茶买孩子的财商教练"。
- **1–3 天试用偏短**（行业常见 7 天）。短试用→习惯未成形即触墙→续费弱。建议试用结构调整（§5）。

**差异化白地（本产品可独占的卖点）**
1. **行为金融实时教练**（`adaptive-events.ts` 已具雏形）——多数竞品只教"知识点"，不诊断"你刚才为什么手痒"。这是**最强护城河**，应放大为核心卖点。
2. **AI 个性化"投资人格/行为评定"** + 可分享的**家长成长报告**——把"学习"转为"可炫耀的成长资产"，驱动家长续费。
3. **可重玩的随机事件引擎**（待补，§5）——把"固定剧本"变"每局不同的财经小剧场"，是留存/重玩的引擎。

## 4. 产品策略：留存-续费-差异化主线

把产品价值主张收敛为一句话：**"不是教你炒股，是用 AI 当镜子，照出并矫正你的投资行为习惯。"**

三条增长飞轮：
1. **价值前置飞轮**：新手村结束就让学生体验一次"完整 AI 行为诊断 + 投资人格卡"（满血），制造 Aha-moment，再谈付费。
2. **家长续费飞轮**：每周/每局生成**家长成长报告**（孩子学了哪些概念、改了哪些坏习惯、行为雷达变化）→ 到期前推送 → 家长为"看得见的进步"续费。
3. **重玩留存飞轮**：随机事件 + 赛季/排行榜/连胜 streak → D1/D7/D30 留存与重玩。

## 5. 具体优化与开发方案（按需求，含文件 / 验收标准）

> 标注优先级 P0(交付前必做) / P1(首个迭代) / P2(增长迭代)。每项给出落点文件与验收标准（AC）。

### 5.1 需求④ —— 随机金融事件引擎（最高价值，P0/P1）

**E1 [P0] 引入种子化随机事件选择**
- 落点：`src/lib/simulation.ts`（事件选择）、`src/lib/market-data.ts`（按 stage/tier 分桶）、`ScenarioRun` 增 `seed` 字段（`src/lib/types.ts`）。
- 设计：每局生成/存储一个 `seed`（复用 `utils.ts` 的 id 思路，但用**确定性 PRNG**如 mulberry32，避免 `Math.random` 破坏可复现与测试）。回合事件从该 stage 的事件桶中**按种子抽取**，黑天鹅按低概率（如后段每回合 8-12%）触发。
- AC：同一 seed 完全可复现（单测）；不同 seed 产生不同事件序列；R1-4 不出黑天鹅；12 回合事件不重复（除非桶不足）。复盘/历史按 seed 重建一致。

**E2 [P0] 让事件真正驱动行情（机制耦合）**
- 落点：`simulation.ts` 回合结算。
- 设计：资产涨跌 = `基础回合漂移 × 事件影响(impactAssets×impactRange×signal方向) × 种子噪声`。即把现 `assetMultipliers` 拆为"基线 + 事件调制 + 随机波动"，让 `impactAssets/impactRange` 真正参与（它们已在数据里，只是没接线）。
- AC：利空事件对其 `impactAssets` 统计上为负向；`impactRange:high` 的波动幅度显著大于 low；存在受益/受损分化（教"相关性/对冲"）。

**E3 [P1] 事件选择/分支玩法（进阶）**
- 设计：部分事件升级为**决策卡**——"流动性危机来了：A 抛售保命 / B 抄底 / C 不动"，不同选择不同后果 + 不同 `teachingConcept`。引入**进阶可操作机制**：杠杆(融资)、对冲(反向/商品)、做空（受限、教风险），承接需求"更进阶的知识与玩法"。
- AC：至少 6 张决策卡含 2-3 分支与差异化结算；杠杆/对冲在 UI 有风险提示与亏损放大演示。

**E4 [P1] 事件库扩容 + 难度三档（衔接 §6 事件库）**

## 附录 B：随机金融事件 — 教学法研究补充（CLT 落地细则）

> 由专项研究综合（同样因 web 被禁用，基于成熟学术与历史知识，URL 需上线前 link-check）。直接对接 §5.1 / §6，可作为 E1-E4 的实现依据。

**B-1 公平性是"随机事件"成败的关键（务必遵守）**
- **种子化 PRNG**：`eventForRound = pick(cardsForTier(round), seededRng(runSeed, round))`，每局存 `runSeed`。好处：可复现（教师可让全班复盘同一剧本）、可测试、可审计。
- **结果要"技能响应"而非"纯抛硬币"**：黑天鹅崩盘要让"已分散/已留安全垫"的玩家少亏 —— 让结果取决于**玩家组合**，体验是"我准备得够不够"而非"我运气差"。纯 RNG 不教任何东西且令人沮丧。
- **怜悯/连负保护**：限制连续负面事件；崩盘后必给 1 张"机会卡"，让结论是"危机=买点"而非"游戏针对我"。
- **预告（telegraphing）**：高阶回合给"风险计上升/不祥头条"的弱提示，教"提前准备"而非"惩罚突袭"。

**B-2 脚手架三段式（CLT 核心，降本增效）**
- **worked-example → 渐隐引导 → 独立决策**：概念首现回合给完整范例（"你满仓 1 股 →跌40%；分散 10 股的玩家只跌6%"）；下一回合"你来补全"；再下回合独立做。
- **专长反转效应**：教程帮新手却拖累熟手 → 微课/范例**只在"首次遇到"时显示**，用 run state 的 `firstSeenLessonShown` flag 门控。
- **拆分注意力/冗余**：headline + 影响 + 解释**同卡**呈现，别一处头条一处 tooltip；同一数字别三种说法重复。

**B-3 12 回合难度曲线（建议基线，覆盖现有写死剧本）**

| 回合 | tier | 新概念 | 锚定事件 |
|---|---|---|---|
| 1 | 1 | 风险/收益、现金也是选择 | 平稳市 + 小额分红 |
| 2 | 1 | 复利/时间价值 | 存款利息入账 |
| 3 | 1 | 分散 | 单股暴雷而大盘稳 |
| 4 | 1→2 | 总回报、拆股 | 分红 + 股票拆分 |
| 5 | 2 | 通胀：现金不安全 | 通胀数据侵蚀现金 |
| 6 | 2 | 利率↔债券/贷款 | 央行加息 |
| 7 | 2 | 杠杆（双刃） | 加杠杆要约 + 范例 |
| 8 | 2→3 | 板块轮动/相关性 | "伪分散"揭示 |
| 9 | 3 | 波动与流动性 | 石油冲击/价差走阔 |
